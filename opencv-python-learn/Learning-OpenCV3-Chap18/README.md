# 18장 카메라 모형과 보정

> 시각(vision)은 실세계에서 빛을 탐지하는 것으로 시작한다. 빛은 광원(예: 전구 또는 태양)에서 퍼져 나와 물체에 부딪힐 때까지 공간을 여행한다. 빛이 물체에 부딪칠 때 많은 빛이 흡수되는데, 물체의 색으로 인식되지 않는 것은 흡수되지 않는다. 우리의 눈 또는 카메라에 도달한 반사된 빛은 망막(또는 영상)에 수집된다. 실제 전산시각에서 물체로부터 눈이나 카메라의 렌즈를 거쳐 망막이나 영상까지 전달되는 빛의 이동에 대한 배치의 기하학은 매우 중요하다.

> 이 과정을 설명하기 위한 간단하지만 유용한 모형으로 **핀홀 카메라 모형(pinhole camera model)** 이 있다. 핀홀은 중심에 작은 구멍이 있는 가상의 벽으로 중앙의 작은 구멍을 통과하는 빛을 제외한 모든 빛을 차단한다. 이 장에서는 빛 투영의 기초 기하학을 이해하기 위한 핀홀 카메라 모형에 대해 다룰 것이다. 아쉽게도 실제 핀홀은 빠른 노출을 위해 충분한 빛을 모으지 않기 때문에 영상을 만드는 좋은 방법은 아니다. 그래서 눈과 카메라는 더 많은 빛을 모으기 위해 렌즈를 사용한다. 그러나 렌즈로 더 많은 빛을 모으는 방법은 핀홀 모형의 단순한 기하학으로는 설명이 불가능할 뿐만 아니라 렌즈 자체에서 왜곡이 발생한다는 단점이 있다.

> 먼저 카메라 모형과 렌즈 왜곡의 원인을 살펴보자. 여기에서는 카메라의 기본 동작과 다양한 왜곡 및 수정에 의한 효과를 찾아낼 수 있는 수학적 도구인 **호모그래피 변환(homography transform)** 을 살펴본다.

---

## 18.1 카메라 모형

> 먼저 카메라의 가장 단순한 모형인 핀홀을 살펴보자. 이 모형에서는 빛이 단순히 장면이나 먼 물체에서 들어오되, 그 장면의 특정 지점에서 단 하나의 광선만 핀홀에 들어오는 것으로 가정한다. 물리적인 핀홀 카메라에서는 이 지점이 촬영 영상 위에 '투영'된다. 결과적으로 이 영상 평면(투영평면이라고도 함)의 영상이 항상 또렷하다(초점이 맞추어 있다). 영상의 크기는 떨어져 있는 대상의 크기를 기준으로 카메라의 단일 변수인 초점 거리에 의해 정해진다. 이상적인 핀홀 카메라의 경우, 핀홀 구멍에서 영상 평면까지의 거리가 정확한 초점거리다.

아래 그림(Fig 1)에서 각 기호의 의미는 다음과 같다.

| 기호 | 의미                              |
| ---- | --------------------------------- |
| $f$  | 초점 거리 (focal length)          |
| $Z$  | 카메라(핀홀)에서 대상까지의 거리  |
| $X$  | 대상의 실제 크기                  |
| $x$  | 영상 평면에 맺힌 상의 크기 (도립) |

**[Fig 1] 핀홀 카메라 모형 — 전통적 표현 (영상이 핀홀 뒤에 맺힘)**

```
     물체 평면             핀홀               영상 평면
          │                 │                    │
          │                 │                    │  ← 광학축
    ──────┤                 │               ─────┤
     A    │                 │                    │
     ┊ X  │ ────────────────●                    │
     ┊    │                  ╲                   │
    ──────┤                   ╲              ─────┤
          │                    ╲─────────── A'   │
          │                    │             ┊ x │ (도립상)
          │                    │                 │
          │◄──────── Z ────────►◄───── f ────────►│

  닮은꼴 삼각형 관계:  -x/f = X/Z
```

$$
-\frac{x}{f} = \frac{X}{Z}
$$

---

> 그림 Fig 2처럼 핀홀 평면과 영상 평면을 맞교환한다. 주요 차이점은 대상이 이제 오른쪽에 나타난다는 점이다. 핀홀의 점(구멍)은 **투영의 중심(projection center)** 으로 재해석된다. 이렇게 사물을 보면 모든 광선이 멀리 떨어진 대상의 한 점에서부터 출발해 투영의 중심을 향해 나아간다. 영상면과 광학축의 교차점을 **주점(principal point)** 이라고 한다. 이 새로운 정면 영상평면은 이전의 투영 평면 또는 영상평면과 동일하다. 따라서 이 새로운 평면에 나타나는 대상의 영상은 Fig 1의 영상 평면에 나타나는 것과 정확히 같은 크기가 된다. 이 광선이 영상 평면과 교차되어 영상이 생성된다. 이 영상은 투영 중심으로부터 정확히 거리 $f$만큼 떨어진 곳에 생성된다. 이것은 닮은꼴 삼각형 관계 $x/f = X/Z$를 전보다 더 직접적으로 나타낸다.

**[Fig 2] 핀홀 카메라 모형 — 정면 영상 평면 표현 (수학적 등가 모형)**

```
     영상 평면          투영 중심              물체 평면
          │              (핀홀)                    │
          │                 │                      │
    ──────┤                 │                 ──────┤
     A'   │                 │                  A    │
     ┊ x  ├─────────────────●──────────────────┊ X  │
          │                 │                       │
    ──────┤                 │                 ──────┤
          │                 │                       │
          │◄───── f ────────►◄──────── Z ────────────►│

  주점(Principal Point): 광학축과 영상 평면의 교점
  닮은꼴 삼각형 관계:   x/f = X/Z  (영상이 뒤집히지 않음)
```

Fig 1과 달리 영상이 정립(正立)으로 나타나며, 동일한 닮음비 $x/f = X/Z$ 가 성립한다.

---

> 주점이 영상면의 중심과 같으려면, 사람이 핀셋과 접착제로 카메라에 영상센서(imager)를 미크론 단위의 정확도로 부착할 수 있어야 한다. 사실, 일반적으로 영상센서 칩의 중심은 광학축 위에 있지 않다. 따라서 투영 화면에 있는 중심 좌표의 편위(광학축에서부터 멀어지는 방향)를 모형화하기 위해 두 개의 새로운 변수 $c_x, c_y$를 도입한다. 그 결과 $(X,Y,Z)$에 위치한 물리적 세계의 점 $\vec{Q}$는 $(x_s, y_s)$에 위치한 촬영 장치의 화소에 다음과 같이 투영되는 비교적 단순한 모형을 얻을 수 있다.

$$
x_s = f_x \frac{X}{Z} + c_x \qquad y_s = f_y \frac{Y}{Z} + c_y
$$

아랫첨자 `s`는 **sensor(센서)** 의 약자로, 이 좌표가 카메라 이미지 센서 위의 픽셀 위치임을 의미한다.

쉽게 말하면, 3D 세계의 점 $(X, Y, Z)$을 카메라로 찍었을 때 **실제 이미지 센서(촬영 소자) 위의 어느 픽셀에 찍히는지** 를 나타내는 좌표다.

---

**비유로 이해하기**

| 좌표         | 의미                       | 단위     |
| ------------ | -------------------------- | -------- |
| $(X, Y, Z)$  | 현실 세계의 점 위치        | m, cm 등 |
| $(x_s, y_s)$ | 카메라 사진 위의 픽셀 좌표 | pixel    |

예: 눈앞에 사과가 있다면

- 사과의 실제 위치 → $(X, Y, Z)$
- 카메라로 찍었을 때 사진의 **300번째 열, 200번째 행 픽셀** → $(x_s, y_s) = (300, 200)$

> `s` 첨자의 핵심 의미: **"이건 렌즈를 통해 센서판 위에 맺힌 좌표야"** 를 강조하는 표시

수식 $x_s = f_x \dfrac{X}{Z} + c_x$ 는 **"현실의 점을 픽셀 좌표로 변환하는 공식"** 이다.

## 카메라 내부 변수 직관적 설명

### 상황 설정

눈앞 1m 거리에 친구가 서 있고, 그걸 카메라로 찍는다고 상상해보자.

---

### $f_x$, $f_y$ — 확대 배율 ("줌 배율")

현실 물체가 카메라 픽셀 좌표에 얼마나 크게 찍히는지를 결정하는 값.

친구가 현실에서 1m 옆으로 이동했을 때 사진에서 **몇 픽셀** 움직여 보이냐를 결정한다.

| $f_x$ 값 | 현실 1m 이동 시 사진에서 이동량 |
| -------- | ------------------------------- |
| 500      | 500픽셀 이동                    |
| 1000     | 1000픽셀 이동 (더 확대된 느낌)  |

$f_x$와 $f_y$가 **따로 존재하는 이유**: 저가 카메라는 픽셀이 정사각형이 아니라 직사각형이라서 가로/세로 방향 확대 비율이 미묘하게 다를 수 있기 때문이다.

> 실제로는 $f_x = f \cdot \dfrac{1}{s_x}$, $f_y = f \cdot \dfrac{1}{s_y}$ 형태이지만, 카메라를 분해하지 않고서는 $f$와 $s_x$를 개별적으로 측정할 수 없으므로 두 값의 곱인 $f_x$, $f_y$ 만을 보정(calibration)으로 구한다.

---

### $c_x$, $c_y$ — 주점 위치 ("사진의 광학 중심")

카메라 렌즈의 광학적 중심이 **사진의 정중앙에 딱 맞지 않을 수 있다.** 공장에서 렌즈와 센서를 조립할 때 미세하게 어긋나기 때문이다.

```
이상적인 경우:           실제:
┌──────────┐            ┌──────────┐
│          │            │          │
│    ✕     │            │      ✕   │  ← 중심이 살짝 오른쪽으로 밀림
│  (중심)   │            │    (중심) │
└──────────┘            └──────────┘
c_x = 사진너비/2         c_x = 사진너비/2 + 살짝
```

> $c_x, c_y$는 **렌즈 광학 중심이 픽셀 기준으로 어느 위치에 있는지** 를 나타내는 값

---

### 카메라 내부 파라미터(Intrinsic Parameters) 요약

| 변수  | 직관적 의미                     |
| ----- | ------------------------------- |
| $f_x$ | 가로 방향 줌 배율 (픽셀 단위)   |
| $f_y$ | 세로 방향 줌 배율 (픽셀 단위)   |
| $c_x$ | 렌즈 광학 중심의 가로 픽셀 위치 |
| $c_y$ | 렌즈 광학 중심의 세로 픽셀 위치 |

이 4개를 묶어서 **카메라 내부 파라미터(intrinsic parameters)** 라고 부르며, 카메라마다 고유한 값이다. **카메라 보정(calibration)** 을 통해 이 값들을 측정한다.

## 18.1.1 투영기하학의 기초

> 실제 세계의 좌표 $(X_i, Y_i, Z_i)$에 있는 점들의 그룹 $\vec{Q}$를 투영화면의 좌표 $(x_i, y_i)$에 있는 점들로 사상(mapping)하는 과정을 **투영변환(projective transform)** 이라고 한다. 이런 변환 작업을 할 때는 **동차좌표(homogeneous coordinate)** 를 사용하는 것이 편리하다.

> n차원의 투영공간에 있는 한 점에 대한 동차좌표는 일반적으로 n+1차원의 벡터로 표현된다. (예: $(x,y,z)$는 $(x,y,z,w)$가 된다.) 여기서는 영상평면이 투영공간이고 2차원을 가지므로, 그 평면상의 점을 3차원 벡터 $\vec{q} = (q_1, q_2, q_3)$로 나타낸다. 투영 공간에서 비례하는 값을 갖는 모든 점들이 동일하다는 것을 생각하면 $q_3$로 나눠 실제 화소좌표를 얻을 수 있다. 이렇게 하면 카메라를 정의하는 변수(즉, $f_x, f_y, c_x, c_y$)를 하나의 3×3 행렬로 나타낼 수 있다. 이를 **카메라 내부변수 행렬** 이라고 부른다. 실제 세계의 점을 카메라에 투영한 결과는 다음과 같은 간단한 형태로 요약된다.

$$
\vec{q} = M\vec{Q}
$$

$$
\vec{q} = \begin{bmatrix} x \\ y \\ z \end{bmatrix}, \quad
M = \begin{bmatrix} f_x & 0 & c_x \\ 0 & f_y & c_y \\ 0 & 0 & 1 \end{bmatrix}, \quad
\vec{Q} = \begin{bmatrix} X \\ Y \\ Z \end{bmatrix}
$$

행렬 $M$을 **카메라 내부변수 행렬(Camera Intrinsic Matrix)** 이라고 한다.

> 여기서 $q=Z$이고 점 $\vec{q}$는 동차좌표이기 때문에 이전의 정의처럼 $w$(또는 $Z$)로 나누면 실제 좌표를 얻을 수 있다. 이제 핀홀 뒤에 있는 투사화면상 거꾸로 된 영상이 아니라 핀홀 앞에 있는 투영평면상 반전되지 않은 영상을 보고 있기 때문에 음수 부호는 사라진다.

### 🤔 왜 동차좌표를 쓰는가?

핵심 문제는 이거예요. 3D 점 $(X, Y, Z)$를 2D 픽셀 $(x_s, y_s)$로 변환하는 공식이:

$x_s = f_x \frac{X}{Z} + c_x$

이렇게 **나눗셈**이 포함되어 있어요. 나눗셈은 행렬 곱셈으로 표현이 안 돼요. 그래서 수학자들이 꼼수를 고안했는데, 그게 바로 **동차좌표**예요.

---

### 💡 동차좌표의 아이디어

나눗셈을 나중으로 미루는 거예요.

**일단 나누지 말고** 행렬 곱셈을 먼저 한 다음, **마지막에 한 번만 나누자!**

> 관련해서 OpenCV 라이브러리 함수인 `cv::convertPointsToHomogeneous()`와 `cv::convertPoints
FromHomogeneous()` 함수를 사용하면 동차좌표로 변환하거나, 동차좌표에서 변환할 수 있다.

> 다음은 이 함수들의 원형(prototype)이다.

```java
void cv::convertPointsToHomogeneous(
cv::InputArray src, // N 차원의 점에 대한 입력 벡터
cv::OutputArray dst // N+l 차원의 점에 대한 결과 벡터
);
void cv::convertPointsFromHomogeneous(
cv::InputArray src, // N 차원의 점에 대한 입력 벡터
cv::OutputArray dst // N-l 차원의 점에 대한 결과 벡터
);
```

> 첫번째 함수는 (일반적인 표현에서) $N$ 차원 점들의 벡터를 입력받아서 그 벡터로
> 부터 $N+1$ 차원의 벡터를 만들어낸다. 추가된 차원과 연관된 새로 구성된 벡터의 모든 요소는 1로 설정된다. 그 결과는 다음과 같다.

$\vec{D_i} = (s_{i,0}, s_{i, 1}, ..., s_{i, N-1}, 1)$

> 두번째 함수는 동차좌표로부터 다시 변환한다. $N$ 차원 점들의 입력벡터가 주어지면 먼저 각 점의 모든 구성요소를 마지막 구성요소값으로 나눈후, 1이 된 그 마지막 구성요소를 버리고 $N-1$차원 점들의 벡터를 만든다. 그 결과는 다음과 같다.

$\vec{D_i} = (\frac{s_{i,0}}{s_{i, N-1}}, \frac{s_{i,1}}{s_{i, N-1}}, ..., \frac{s_{i,N-2}}{s_{i, N-1}})$

> 이상적인 핀홀을 사용하면 비전 분야에서는 일부 3차원 기하학을 위한 유용한 모형을 만들 수 있다. 그러나 실제로는 핀홀을 통과하는 빛이 거의 없어 어떤 센서를 사용하든 충분한 빛이 축적될 때까지 기다려야 하기 때문에 그러한 배치로는 촬영이 매우 오래 걸린다. 카메라가 더 빠른 속도로 영상을 획득하려면 더 넓은 영역에
> 서 많은 빛을 모으고 빛이 투사 지점에서 수렴되도록 빛을 굴절시켜야 한다(즉, 빛
> 을 투사지점에 집중시켜야 한다.)
> 이러한 용도로 렌즈가 사용된다. 렌즈는 한 지점에 많은 양의 빛을 집중시켜 빠르게 촬영할 수 있지만, 그로 인해 발생하는 왜곡은 감수해야 한다.

### 18.1.2 로드리게스 변환

> 3차원 공간을 다룰 때는 공간에서 3x3 행렬로 표현한 회전을 주로 사용한다. 벡터를 이 행렬에 곱하는 것과 벡터를 특정한 방향으로 회전시키는 것이 서로 같기 때문에 보통 이 표현 방식이 가장 편리하다. 단점은 어느 3x3 행렬이 어떤 회전을 하는지 직관적으로 알기 어렵다는 것이다. 따라서 이 장에서는 일부 OpenCV 함수에서 사용되는 회전에 대한 대체 표현과 이 대체 표현으로 변환하는 유용한 함수를 소개한다.

---

#### 축-각도 표현 (Axis-Angle Representation) 직관

3D 공간에서 어떤 회전이든 **하나의 축(axis)과 그 축 주위의 회전 각도(angle)** 로 표현할 수 있다.

```
        z
        │   ← 회전축 방향 (단위벡터 r_hat)
        │  ↗
        │ /  θ = 회전 각도 (벡터의 크기)
        │/
        └──────── x
       /
      y
```

로드리게스 벡터 $\vec{r}$은 이 두 정보를 **하나의 벡터에 압축**한다.

$$
\vec{r} = \theta \cdot \hat{r}
$$

| 의미 | 수식 | 설명 |
| ---- | ---- | ---- |
| 회전 각도 | $\theta = \|\vec{r}\|$ | 벡터의 크기(길이) |
| 회전 축 방향 | $\hat{r} = \dfrac{\vec{r}}{\|\vec{r}\|}$ | 벡터를 정규화한 단위벡터 |

> 예: $\vec{r} = \begin{bmatrix} 0 \\ 0 \\ \pi/2 \end{bmatrix}$ 이면, z축 주위로 90° 회전을 의미한다.

---

#### 두 가지 회전 표현 비교

| 표현 방식 | 형태 | 장점 | 단점 |
| --------- | ---- | ---- | ---- |
| **회전 행렬** $R$ | 3×3 행렬 (9개 원소) | 계산이 빠름, 벡터에 바로 곱할 수 있음 | 직관적으로 이해하기 어려움 |
| **로드리게스 벡터** $\vec{r}$ | 3×1 벡터 (3개 원소) | 직관적, 저장 공간 작음 | 행렬 연산 불가, 변환 필요 |

---

#### 벡터 → 회전 행렬 변환 공식

> $\vec{r} = \begin{bmatrix} r_x & r_y & r_z \end{bmatrix}^T$, $\theta = \|\vec{r}\|$, $\hat{r} = \vec{r}/\theta$ 로 정의한다.

$$
R = \cos\theta \, I_3 + (1-\cos\theta)\,\hat{r}\hat{r}^T + \sin\theta \begin{bmatrix} 0 & -\hat{r}_z & \hat{r}_y \\ \hat{r}_z & 0 & -\hat{r}_x \\ -\hat{r}_y & \hat{r}_x & 0 \end{bmatrix}
$$

**수식 3항 분해:**

| 항 | 역할 |
| --- | --- |
| $\cos\theta \, I_3$ | 회전 전 기존 성분 유지 (cosine 비율) |
| $(1-\cos\theta)\,\hat{r}\hat{r}^T$ | 회전 축 방향 성분 보정 |
| $\sin\theta [\hat{r}]_\times$ | 축 주위로 회전시키는 성분 (반대칭 행렬) |

> 마지막 항의 $\begin{bmatrix} 0 & -\hat{r}_z & \hat{r}_y \\ \hat{r}_z & 0 & -\hat{r}_x \\ -\hat{r}_y & \hat{r}_x & 0 \end{bmatrix}$ 은 **반대칭 행렬(skew-symmetric matrix)** 로, 외적(cross product) 연산을 행렬로 표현한 것이다.

---

#### 회전 행렬 → 벡터 역변환 공식

회전 행렬 $R$에서 축-크기 표현으로 되돌리려면 $R$의 반대칭 부분을 이용한다.

$$
\sin\theta \begin{bmatrix} 0 & -\hat{r}_z & \hat{r}_y \\ \hat{r}_z & 0 & -\hat{r}_x \\ -\hat{r}_y & \hat{r}_x & 0 \end{bmatrix} = \frac{R - R^T}{2}
$$

$$
\theta = \arccos\left(\frac{\mathrm{trace}(R) - 1}{2}\right)
$$

> trace$(R)$는 행렬의 대각 원소의 합, 즉 $R_{00} + R_{11} + R_{22}$이다.

---

#### 주요 성질

- 회전 행렬 $R$은 항상 **직교 행렬(orthogonal matrix)**: $R^T R = I$
- $\det(R) = +1$ (반사 없이 순수 회전)
- $\vec{r} = \vec{0}$ (영벡터)이면 $R = I$ (항등 변환, 회전 없음)
- $\vec{r}$과 $-\vec{r}$은 반대 방향 회전 (반시계 ↔ 시계)

---

#### OpenCV 함수: `cv::Rodrigues()`

> 따라서 계산에 가장 편리한 표현(행렬 표현)과 사람의 뇌가 이해하기에 좀 더 쉬운 표현(로드리게스 표현)을 사용할 수 있다. OpenCV는 표현끼리 서로 변환할 수 있는 함수를 제공한다.

```cpp
void cv::Rodrigues(
    cv::InputArray src,                       // 입력: 회전 벡터(3×1) 또는 회전 행렬(3×3)
    cv::OutputArray dst,                      // 출력: 회전 행렬(3×3) 또는 회전 벡터(3×1)
    cv::OutputArray jacobian = cv::noArray()  // (선택) 자코비안 행렬 (3×9 또는 9×3)
);
```

**입출력 방향:**

| `src` | `dst` | 변환 방향 |
| ----- | ----- | --------- |
| 3×1 회전 벡터 $\vec{r}$ | 3×3 회전 행렬 $R$ | 벡터 → 행렬 |
| 3×3 회전 행렬 $R$ | 3×1 회전 벡터 $\vec{r}$ | 행렬 → 벡터 |

> `jacobian` 인수가 `cv::noArray()`가 아니면, 입력 배열의 각 원소에 대한 출력 배열의 편미분으로 채워진 3×9 또는 9×3 자코비안 행렬을 반환한다. 이는 최적화 알고리즘(예: 카메라 보정)에서 활용된다.

---

#### Python 예제

```python
import cv2
import numpy as np

# 1) 로드리게스 벡터 → 회전 행렬
rvec = np.array([[0.0], [0.0], [np.pi / 2]])  # z축 주위로 90° 회전
R, jacobian = cv2.Rodrigues(rvec)
print("회전 행렬 R:")
print(np.round(R, 4))
# [[ 0. -1.  0.]
#  [ 1.  0.  0.]
#  [ 0.  0.  1.]]

# 2) 회전 행렬 → 로드리게스 벡터 (역변환)
rvec_back, _ = cv2.Rodrigues(R)
print("\n역변환 결과 rvec:")
print(np.round(rvec_back, 4))
# [[0.    ]
#  [0.    ]
#  [1.5708]]  ← π/2 ≈ 1.5708
```

---

#### 실제 활용: 카메라 보정의 외부 파라미터

카메라 보정(`cv2.calibrateCamera()`)의 결과로 반환되는 `rvecs`가 바로 로드리게스 벡터다.

```python
ret, mtx, dist, rvecs, tvecs = cv2.calibrateCamera(
    objpoints, imgpoints, gray.shape[::-1], None, None
)
# rvecs[i]: i번째 보정 영상에서 카메라와 체스보드 간의 상대 회전 (로드리게스 벡터)
# tvecs[i]: i번째 보정 영상에서 카메라와 체스보드 간의 상대 이동 (평행이동 벡터)

# 회전 행렬로 변환
R, _ = cv2.Rodrigues(rvecs[0])
```

> 로드리게스 벡터는 **저장 및 최적화 단계에서 3개의 값**만 다루면 되므로, 9개 원소의 회전 행렬보다 훨씬 효율적이다.

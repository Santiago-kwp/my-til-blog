# 6. 화소 처리

> 화소란 화면(영상)을 구성하는 가장 기본이 되는 단위를 말한다.
> 디지털 영상은 이 화소들의 집합을 의미하며, 이 화소들에 대해 다양한 연산을 하는 것이 영상처리이다.

## 6.1 영상 화소의 접근

> 영상처리를 아주 간단하게 말해보면, 2차원 데이터에 대한 행렬 연산이라고 할 수 있다.
> 또한, 영상처리 프로그래밍을 한다는 것은 영상이라는 2차원 데이터의 원소값을 개발자가 원하는 방향으로 변경하는 것이라고 할 수 있다.
> OpenCV API도 numpy.ndarray 객체를 기반으로 영상 데이터를 처리한다.

### 6.1.1 화소(행렬 원소) 접근

> `01.mat_access.py`, `02.image_access.py` 참조
> 행렬 화소에 직접 접근하는 경우 수행속도가 엄청 느리므로, OpenCV에서 제공하는 함수들을 조합하거나 ndarray 객체의 원소간 연산으로 구현 내용을 만드는 것을 권장한다.

## 6.2 화소 밝기 변환

### 6.2.1 그레이 스케일(명암도) 영상

> 디지털 영상처리에서 보통 단일채널의 영상을 그레이 스케일(gray-scale) 영상이라고 부른다. 우리말로는 명암도 영상이라 한다.

하나의 화소값은 0~255의 값을 가지는데 0은 검은색을, 255는 흰색을 의미한다.

### 6.2.3 영상 밝기의 가감 연산

> 영상의 화소에 특정한 상숫값을 더하면 영상이 밝아지고, 상숫값을 빼면 영상이 어두워진다. 또한 화소가 가질 수 있는 최댓값에서 그 화소의 값을 빼면 반전 영상이 만들어진다.

> OpenCV의 saturation 방식과 numpy의 modulo 방식의 차이를 이해하자

- OpenCV: 250 + 100 = 350 -> 255 (saturation)
- numpy : 250 + 100 = 350 % 256 -> 104 (modulo)

👉 그래서 영상 밝기/대비 조절 같은 작업에는 OpenCV 함수를 쓰는 게 더 직관적이고 안전하다.

### 6.2.4 행렬 덧셈 및 곱셈을 이용한 영상 합성

> 두 개의 행렬을 합하게 되면, saturation 연산으로 인해서 255를 넘어서는 화소들은 흰색으로 나타나서 영상의 합성이 제대로 수행되지 않는다. 이 문제에 대한 해결 방법은 다음과 같이 여러 가지가 있을 수 있다.

1. dst(y,x) = image1(y,x) _ 0.5 + image2(y,x) _ 0.5;
2. dst(y,x) = image1(y,x) _ alpha + image2(y,x) _ (1-alpha)

3) dst(y,x) = image1(y,x) _ alpha + image2(y,x) _ beta

### 6.2.5 명암 대비

> 명암 대비(constrast)는 상이한 두 가지 색(밝기)이 경계에서 서로 영향을 미쳐 그 차이가 강조되어 나타나는 현상을 말한다.
> 높은 명암대비 영상은 어두운 부분과 밝은 부분이 대비를 이루어 전체적으로 영상이 또렷해 보인다.

> 명암 대비를 늘리기 위해서는 1.0 이상의 값을 곱해주면 되고, 줄이기 위해서는 1.0 이하의 값을 곱해주면 된다.

1. noimage 더미 영상이 굳이 필요한가요?

"이 함수들을 쓰려면 억지로 넣어야 하지만, 사실 더 좋은 대체 함수가 있어서 굳이 이렇게 안 짜도 됩니다."
코드에서 사용한 `cv2.scaleAdd`와 `cv2.addWeighted`는 본래 두 개의 영상을 합성할 때 쓰는 함수입니다.
`cv2.scaleAdd(src1, alpha, src2): (src1 * alpha) + src2` 를 계산합니다.
`cv2.addWeighted(src1, alpha, src2, beta, gamma): (src1 * alpha) + (src2 * beta) + gamma` 를 계산합니다.
작성자분은 영상을 두 개 합치려는 게 아니라, 단지 원본 영상에 숫자를 곱하고 더해서(명암 대비 조절) 결과를 얻고 싶었을 뿐입니다. 하지만 위 함수들은 규칙상 무조건 '두 번째 영상(src2)'을 입력받아야만 작동합니다.
그래서 어쩔 수 없이 아무 영향도 주지 않는 **검은색 빈 도화지(noimage)**를 만들어서 억지로 끼워 넣은 것입니다. (일종의 꼼수입니다.)
💡 더 깔끔한 해결책 (더미 영상 없이 구현하기)
OpenCV에는 딱 이런 명암/밝기 조절을 위해 만들어진 **cv2.convertScaleAbs**라는 함수가 있습니다. 이 함수를 쓰면 noimage를 아예 안 만들어도 됩니다.

```Python
# 더미 영상(noimage) 없이 구현하는 더 좋은 방법
dst1 = cv2.convertScaleAbs(image, alpha=0.5, beta=0)      # dst1과 동일
dst2 = cv2.convertScaleAbs(image, alpha=2.0, beta=0)      # dst2와 동일
dst3 = cv2.convertScaleAbs(image, alpha=0.5, beta=avg)    # dst3과 동일
dst4 = cv2.convertScaleAbs(image, alpha=2.0, beta=-avg)   # dst4와 동일
```

## 6.3 히스토그램

### 6.3.1 히스토그램 개념

> 영상처리에서도 히스토그램은 화소의 분포를 나타내는 지표이기 때문에 이 분포를 이해하면 영상의 특성을 판단할 수 있는 유용한 도구가 될 수 있음.

### 6.3.3 OpenCV 함수 활용

`cv2.calcHist(images, channels, mask, histSize, ranges[, hist[, accumulate ]])`

> 행렬 원소값의 빈도를 계산한다.

- images: 원본 배열들 - CV_8U, CV_32F 형으로 크기가 같아야 함.
- channels : 히스토그램 계산에 사용되는 차원 목록
- mask : 특정 영역만 계산하기 위한 마스크 행렬 - 입력 영상과 같은 크기의 8비트 배열
- histSize: 각 차원의 히스토그램 배열 크기 - 계급(bin)의 개수
- ranges: 각 차원의 히스토그램의 범위
- accumulate: 누적 플래그 - 여러 배열에서 단일 히스토그램을 구할 때 사용

## 1. 배경 및 목적 (Background & Objective)

카메라의 전후각(Pitch Angle, 전방/후방 기울기)이 달라지면, 동일한 피사체(달력)라도
이미지 내에서 차지하는 픽셀 크기(너비, 높이)가 달라진다.

본 기능은:
1. 전후각 -1°, -10°, 6° 조건에서 촬영된 달력 이미지에서 달력 영역의 픽셀 크기를 측정
2. 전후각 변화에 따른 달력 크기 변화 패턴을 분석
3. 특정 전후각에서 촬영된 이미지를 기준 각도(0° 또는 특정 기준) 대비 크기를 보정하는
   Python 함수를 개발
4. 해당 함수의 정확성을 검증하는 테스트 코드 작성

## 3. 이론적 배경 (Theoretical Background)

### 3.1 투시 투영과 피치각 (수정된 올바른 모델)

카메라 핀홀 모델에서, 카메라가 피치각 θ만큼 기울어진 상태로
수직 평면(달력)을 촬영할 때, 달력이 카메라 높이를 중심으로 대칭 배치된 경우
달력의 픽셀 높이 비율은 다음과 같이 유도된다:

```
h_img(θ) / h_img(0) = 1 / (cos²θ - α²·sin²θ)
```

- θ: 기준각 대비 전후각 (기준각 θ_ref 기준의 상대 각도)
- α = H / (2d): 달력 실제 높이의 절반 / 달력까지의 수평 거리
- h_img: 이미지 상에서의 달력 픽셀 높이

**핵심 성질 (⚠️ 직관적 오해 주의):**

| 구분 | 설명 |
|------|------|
| **대칭성** | 공식은 θ와 -θ에 대해 동일 → 앞으로 기울든 뒤로 기울든 같은 크기 변화 |
| **단조 증가** | 기준각(θ_ref)에서 멀어질수록 cos²θ - α²sin²θ가 작아지므로 비율 ≥ 1 |
| **❌ 틀린 직관** | "뒤로 기울면 크게, 앞으로 기울면 작게" → 이 설명은 잘못된 단순화 |
| **✅ 바른 직관** | **어느 방향이든** 기준각에서 멀어질수록 달력이 더 크게 투영됨 |

**물리적 의미:**
달력이 수직으로 서 있을 때, 카메라가 기준 각도에서 어느 방향으로든 기울면
달력 상단과 하단이 카메라에서 서로 다른 거리에 놓이게 되어
원근 왜곡으로 인해 픽셀 높이가 증가한다.

**예상 크기 순서 (|각도 - θ_ref|가 클수록 크게 보임):**
```
|-10°| = 10° (기준 대비) > |+6°| = 6° > |-1°| = 1°
→ h(-10°) > h(+6°) > h(-1°) 예상
```

> **⚠️ 주의**: 위 공식은 달력이 카메라 높이를 기준으로 수직 대칭인 이상적 케이스이다.
> 실제 셋업에서는 카메라 위치와 달력 중심 높이 차이로 비대칭이 발생할 수 있어,
> **실측 데이터 피팅이 필수**이다.

### 3.2 보정 함수 설계 방향 (수정)

1. **투시 투영 기반 모델** (이론적 접근, 권장):
   ```
   scale(θ) = (cos²θ_ref - α²·sin²θ_ref) / (cos²θ - α²·sin²θ)
   ```
   - α와 θ_ref를 실측 데이터로 추정
   - 단순 `cos(θ)` 단일항 모델은 부정확 (양 끝점 거리 차이를 무시함)

2. **다항 회귀 모델** (데이터 기반 접근):
   - 3개 데이터 포인트로 2차 다항식 피팅
   - 모델 검증 후 이론 모델과 비교

## 6. 구현 접근 방식 (Implementation Approach)

### 6.1 달력 영역 감지 전략

달력은 이미지에서 밝은 흰색 직사각형 객체이다.

**후보 방법:**
1. **엣지 검출 + 윤곽선 찾기**: Canny → findContours → 최대 사각형 윤곽 선택
2. **HSV 임계값 세분화**: 흰색 영역 마스킹 → morphology → bounding box
3. **Adaptive threshold + Hough lines**: 수평/수직선 교점으로 직사각형 추출

**선택**: 방법 1 (Canny + contour) → 간단하고 해석 가능

## 파이썬 문

### 파이썬 3.X 키워드용 인자
> 파이썬 3.X는 키워드용 인자를 허용하는 함수 정의를 보편적으로 사용하는데, 키워드에 의해 전달되어야 하고 기본값으로 코딩되지 않은 경우에 필요하다. 키워드용 인자는 * 다음에 코딩되는데, 키워드용 인자가 있고 임의의 위치 인자가 없을 경우 이름 없이 사용될 수 있다.

```python
def f(a, *b, c): print(a, b, c) # 키워드 c 필요
...
f(1, 2, c=3)
1 (2,) 3

def f(a, *, c=None): print(a, c) # 키워드 c 생략 가능
...
f(1)
1 None
f(1, c='spam')
1 spam
```

> 파이썬 3.X 버전부터는 함수를 정의할 때 '키워드 전용 인자(keyword-only arguments)'라는 특별한 인자를 사용할 수 있게 되었어요. 이름 그대로 이 인자들은 함수를 호출할 때 반드시 '키워드'를 사용해서 값을 전달해야만 하는 인자들을 말합니다.

#### 키워드 전용 인자(Keyword-Only Arguments)는 왜 필요할까요?

>보통 함수를 호출할 때는 인자(argument)의 위치만 맞춰서 값을 전달할 수도 있고, 인자 이름(키워드)을 명시해서 값을 전달할 수도 있습니다.

```python
def greet(name, message):
    print(f"Hello, {name}! {message}")

greet("Alice", "How are you?") # 위치 인자 (Alice는 name, How are you?는 message)
greet(message="Good morning", name="Bob") # 키워드 인자 (명확하죠?)
```

>그런데 특정 인자는 **반드시 키워드를 써서 값을 넘겨주도록 강제하고 싶을 때**가 있어요. 왜냐하면:

1.  **가독성 향상:** 함수의 특정 인자가 어떤 의미인지 명확하게 하고 싶을 때. 특히 인자가 많을 때 키워드를 사용하면 훨씬 읽기 쉬워져요.
2.  **버그 예방:** 인자의 순서를 실수로 바꿔서 잘못된 값이 전달되는 것을 막을 수 있어요.

#### 키워드 전용 인자를 어떻게 만들까요?

파이썬에서는 `*` (별표)를 사용해서 키워드 전용 인자를 구분해요.

**1. `*args` 뒤에 오는 인자들**

함수 정의에서 `*` 뒤에 오는 인자들은 모두 키워드 전용 인자가 됩니다. 여기서 `*args`는 파이썬에서 가변 위치 인자를 받을 때 사용하는 표기법인데, 사실 `*`만 단독으로 사용해도 키워드 전용 인자를 만들 수 있어요.

**예시 1: `def f(a, *b, c):`**

```python
def f(a, *b, c): # 여기서 'c'는 키워드 전용 인자입니다.
    print(a, b, c)

# f(1, 2, 3) # 에러 발생! 'c'는 키워드 없이 전달될 수 없습니다.
f(1, 2, c=3) # 올바른 호출. 'c'에 키워드를 사용했습니다.
# 출력: 1 (2,) 3

f(10, c=20) # 'b'는 빈 튜플이 됩니다.
# 출력: 10 () 20
```

*   여기서 `*b`는 `b`가 가변 위치 인자들을 튜플로 묶어서 받는다는 뜻이에요.
*   `c`는 `*b` 뒤에 나왔기 때문에 **반드시 `c=`와 같이 키워드를 써서 값을 전달해야 합니다.**
*   `f(1, 2, c=3)`에서 `1`은 `a`에, `2`는 `*b`에 포함되어 `(2,)` 튜플이 되고, `c=3`은 `c`에 전달됩니다.

**2. `*`만 단독으로 사용한 경우 (가변 위치 인자 없이 키워드 전용 인자만)**

만약 가변 위치 인자(예: `*b`)는 필요 없고, 단순히 특정 인자들만 키워드 전용으로 만들고 싶다면 `*`만 단독으로 사용할 수 있어요.

**예시 2: `def f(a, *, c=None):`**

```python
def f(a, *, c=None): # 여기서 'c'는 키워드 전용 인자입니다.
    print(a, c)

f(1) # 'a'에 1이 전달되고, 'c'는 기본값인 None을 사용합니다.
# 출력: 1 None

f(1, c='spam') # 'a'에 1이 전달되고, 'c'에 'spam'이 전달됩니다. (키워드 사용)
# 출력: 1 spam

# f(1, 'spam') # 에러 발생! 'c'는 키워드 없이 전달될 수 없습니다.
```

*   `def f(a, *, c=None)`에서 `*`는 **"여기까지는 위치 인자고, 이제부터 나오는 인자들은 모두 키워드 전용 인자다"**라는 표시 역할을 해요.
*   `a`는 일반적인 위치 인자이고, `c`는 `*` 뒤에 나왔기 때문에 키워드 전용 인자가 됩니다.
*   `c=None`처럼 기본값을 설정할 수도 있어요. 이렇게 하면 함수를 호출할 때 `c` 값을 생략해도 에러가 나지 않고 기본값이 사용됩니다.

#### 요약하자면:

*   **키워드 전용 인자**는 함수를 호출할 때 **반드시 인자 이름을 명시(`인자이름=값`)해서 값을 전달해야 하는 인자**입니다.
*   함수 정의에서 `*` (별표) 뒤에 오는 인자들이 키워드 전용 인자가 됩니다.
*   `*`만 단독으로 사용하여 가변 위치 인자 없이 키워드 전용 인자만 지정할 수도 있습니다.
*   이를 통해 함수의 가독성을 높이고, 호출 시 발생할 수 있는 오류를 줄일 수 있습니다.

#### 1. `*args` (가변 위치 인자)

>`*b`와 같이 `*` 뒤에 오는 하나의 변수 이름은 **가변 위치 인자**를 의미합니다. 이것은 함수를 호출할 때 몇 개의 위치 인자가 오든지 상관없이 모두 받아서 튜플 형태로 묶어주는 역할을 합니다.

**예시:**

```python
def sum_all(*numbers): # *numbers는 가변 위치 인자
    print(f"받은 숫자들: {numbers}")
    return sum(numbers)

sum_all(1, 2, 3) # 받은 숫자들: (1, 2, 3)
sum_all(10, 20)  # 받은 숫자들: (10, 20)
```

여기서 `numbers`는 `(1, 2, 3)`이나 `(10, 20)`과 같은 **튜플**이 됩니다. 이 인자들을 호출할 때 키워드(`numbers=...`)로 전달하지 않죠? 그냥 위치에 맞춰서 값을 전달하는 일반적인 위치 인자예요.

#### 2. `*` 뒤에 쉼표로 구분되어 오는 인자 (키워드 전용 인자)

>키워드 전용 인자는 **함수 정의에서 `*` (가변 위치 인자) 또는 단독 `*` 뒤에 쉼표(`,`)로 구분되어 나오는 인자**를 말합니다.

**예시 1: `*b` 뒤에 `c`가 오는 경우**

```python
def f(a, *b, c): # 여기서 'b'는 가변 위치 인자, 'c'는 키워드 전용 인자
    print(f"a: {a}, b: {b}, c: {c}")

f(1, 2, 3, c=4)
# 출력: a: 1, b: (2, 3), c: 4
```

*   여기서 `*b`는 `2, 3`을 튜플 `(2, 3)`으로 받아요.
*   `c`는 `*b` 뒤에 **쉼표로 구분되어 나왔기 때문에** 키워드 전용 인자가 됩니다. 따라서 `c=4`처럼 반드시 키워드를 사용해야 합니다.

**예시 2: 단독 `*` 뒤에 `c`가 오는 경우**

```python
def g(a, *, c): # 여기서 'a'는 일반 위치 인자, 'c'는 키워드 전용 인자
    print(f"a: {a}, c: {c}")

g(1, c=2)
# 출력: a: 1, c: 2
```

*   `*`는 "여기까지는 위치 인자고, 지금부터는 키워드 전용 인자다"라는 경계선 역할을 합니다.
*   `c`는 이 단독 `*` 뒤에 **쉼표로 구분되어 나왔기 때문에** 키워드 전용 인자가 됩니다.

#### 핵심 정리:

*   `*변수이름` (예: `*b`)는 **가변 위치 인자**입니다. 호출 시 키워드 없이 위치에 맞춰 전달됩니다.
*   `*` 또는 `*변수이름` 뒤에 **쉼표(`,`)로 구분되어 나오는 인자**들 (예: `c` in `def f(a, *b, c):` 또는 `c` in `def g(a, *, c):`)이 **키워드 전용 인자**입니다. 이들은 호출 시 반드시 키워드(`c=값`)를 사용해야 합니다.

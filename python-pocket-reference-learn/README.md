## 내장 자료형과 연산자

### 연산자와 우선순위
> 표1은 파이썬의 표현식 연산자를 보여준다. 연산자들이 괄호 없이 혼합되어 사용된 식에서는 표 안에서 아래에 있는 연산자들이 위에 있는 연산자들보다 연산의 우선순위를 가진다.

#### 원자적 용어와 동적 타이핑
> 표 1에서 대체 가능한 표현식 항목인 X, Y, Z, i, j, k는 다음 중 하나일 수 있다.
- 가장 최근에 대입된 값으로 대체되는 변수명
- "파이썬 내장 자료형" 섹션에 정의된 리터럴식
- 이 표의 아무 행에서나 볼 수 있는 중첩식

✏️ 원자적 용어란, 더 이상 연산자로 쪼갤 수 없는 “표현식의 최소 단위”를 말한다.
1. 변수명 (이름, name)
```
x
total
my_value
```
- 가장 최근에 대입된 값을 참조
- 자체로 하나의 완전한 표현식
- 더 쪼갤 수 없음

2. 리터럴(literal)
```
10
3.14
"hello"
True
None
```
- 이미 값 자체
- 연산자를 포함하지 않음

3. 괄호로 묶인 표현식
```
(x + y)
```
- 내부는 복합 표현식이지만
- 겉에서 보면 하나의 단위로 취급

4. 속성 참조, 인덱싱, 호출 등 (문법적으로 원자 취급)
표 1의 아래쪽에 있는 것들입니다:
```python
X[i]        # 인덱싱
X[i:j:k]    # 슬라이싱
X(args)     # 함수 호출
X.attr      # 속성 참조
```
- 더 높은 우선순위
- 문법적으로 “붙어서 하나처럼” 작동


> 파이썬 변수는 동적 타이핑 모델을 따른다. 그것들은 미리 선언되지 않으며, 대입될 때 생성된다. 그리고 값을 가리키는 객체 참조를 가지는데, 어떠한 형태의 객체라도 참조할 수 있다. 또한 기본값을 가지지 않으므로 표현식에 나타나기 전에 반드시 미리 대입돼야(assigned) 한다. 변수명의 대소문자는 항상 구분된다. 변수에 의해 참조된 객체는 자동으로 생성되고, 그것이 더는 사용되지 않을 때 CPython의 참조 카운터를 사용하는 파이썬 가비지 컬렉터에 의해 자동으로 회수된다.

> 표 1에서 대체 가능한 attr은 반드시 한 속성의 리터럴명이어야 한다. args1은 "def문" 섹션에 정의된 대로 형식 인자 리스트이다. args2는 "식 문" 섹션에 정의된 대로 입력 인자 리스트이다. 그리고 리터럴은 3.X에 한해 원자적 표현식으로 사용된다.

> 컴프리헨션의 문법과 표 1 중 마지막 세 행의 데이터 구조 리터럴들(튜플, 리스트, 딕셔너리, 집합)은 "파이썬 내장 자료형" 절을 참고하자

#### 원자적 용어 vs primary

> **원자적 용어(atom)** 는 *표현식의 최소 구성 요소*이고,  
> **primary** 는 *그 atom 위에 인덱싱·속성 접근·호출 등을 덧붙여 만든 표현식*이다.

즉, **atom ⊂ primary**

1. 파이썬 문법에서의 공식 정의

파이썬 레퍼런스 문법에는 다음과 같은 구조가 있습니다:

```text
atom        → NAME | NUMBER | STRING | ... | (expression) | ...
primary     → atom
            | primary "[" slicing "]"
            | primary "." NAME
            | primary "(" [arguments] ")"
            | primary "[" list_display "]"
```

이걸 말로 풀면:

*   **atom**: 더 나눌 수 없는 가장 기본적인 표현식
*   **primary**: atom에 **후행 연산(postfix)** 을 0개 이상 붙인 결과

***

2. atom(원자적 용어)이란?

정의

**atom**은 파이썬 표현식 문법에서  
👉 *더 이상 문법적으로 쪼갤 수 없는 최소 단위*입니다.

atom의 예

```python
x
42
"hello"
True
None
(a + b)
[1, 2, 3]
{1, 2}
```

특징:

*   자체로 완전한 표현식
*   연산자 없이 값이 확정
*   식의 “출발점”

✅ 이 단계에서는 **호출, 인덱싱, 속성 접근이 아직 없음**

***

3. primary란?

정의

**primary**는 atom을 기반으로 해서  
👉 **인덱싱, 슬라이싱, 속성 접근, 호출** 같은 연산을 *연속적으로 적용할 수 있는 표현식*입니다.

primary의 예

```python
x
x[0]
x[0][1]
obj.attr
obj.method()
f(1)(2).attr[0]
```

이 중에서:

*   `x` → atom이자 primary
*   `x[0]` → **atom은 아님**, primary
*   `obj.method()` → primary
*   `f(1)(2).attr[0]` → 계속 확장된 primary

***

4. 핵심 차이 포인트 정리

| 구분     | atom (원자적 용어)     | primary                 |
| ------ | ----------------- | ----------------------- |
| 문법적 위치 | 가장 바닥             | atom **위 단계**           |
| 역할     | 표현식의 최소 단위        | 실제 연산이 붙는 대상            |
| 후행 연산  | ❌ 불가              | ✅ 가능                    |
| 예시     | `x`, `10`, `"hi"` | `x[i]`, `x.attr`, `x()` |
| 중첩 가능성 | ❌ 자체는 단순          | ✅ 무한히 체인 가능             |

***

5. 왜 이런 구분이 필요한가?

예제 1

```python
obj.method()[0]
```

문법적으로 보면:

1.  `obj` → atom
2.  `obj.method` → primary
3.  `obj.method()` → primary
4.  `obj.method()[0]` → primary

👉 atom은 **하나**, primary는 **계속 확장**됨

***

예제 2

```python
(x + y)[0]
```

*   `(x + y)` → atom (괄호 덕분에 하나의 atom)
*   `(x + y)[0]` → primary

👉 그래서 **괄호가 중요한 이유**도 여기서 나옵니다.

***


*   많은 입문서에서
    *   atom, primary, expression
    *   모두 그냥 “표현식”이라고 뭉뚱그려 설명함
*   그래서 primary까지도 “원자적”이라고 느껴지는 경우가 생김

하지만 **엄밀히 말하면**:

*   `x[0]`, `f()` 는 **원자적 아님**
*   대신 **primary**

***


## 파이썬 문
> 이번 절에서는 파이썬 문에 관해 설명한다. 각 문장의 문법 형식 목록을 기입하고, 상세한 용법을 알아본다. 복합문으로 구성된 한 개나 여러 개의 다른 문장을 의미하는 문장 형식의 각 묶음은, 헤더 라인 밑에 블록으로 들여쓰게 될 것이다. 한 묶음이 다른 복합문(if, while 등)을 포함하면 그것은 헤더 밑에서 반드시 들여써야 한다. 그렇지 않으면, 그것은 문장 헤더와 같은 라인에 나타날 수 있다. 다음은 둘 다 유효한 생성이다.
```python
if x < 42:
    print(x)
    while x: x = x-1

if x < 42: print(x)
```

>헤더는 파이썬의 복합문(compound statement) 에서
👉 “이제부터 어떤 블록이 시작된다” 라고 알려주는 문장의 앞부분 전체를 말합니다.
즉,
- 조건식
- 키워드 (if, while, for, def, class 등)
- 마지막의 콜론 :

까지 포함한 논리적인 문장 단위가 헤더입니다.

### 대입문
```python
target = expression
target1 = target2 = expression
target1, target2 = expression1, expression2
target1 += expression

target1, target2, ... = same-length-iterable
(target1, target2, ...) = same-length-iterable
[target1, target2, ...] = same-length-iterable
target1, *target2, ... = matching-length-iterable
```

> 모든 대입은 대상에 객체 참조를 저장한다. 대입문은 앞에서 제시한 명시적 문법 포맷을 통해 대입을 요청한다.

- 표현식은 객체를 생산한다.
- 대상은 단순히 이름(X)일 수도 있고 제한적 속성(X.attr)일 수도 있으며, 인덱스나 슬라이스(X[i], X[i:j:k])일 수도 있다.
- 대상 내의 변수는 미리 선언되지 않지만, 표현식에서 사용되기 전에 대입돼 있어야 한다

### 파이썬 3.X 키워드용 인자
> 파이썬 3.X는 키워드용 인자를 허용하는 함수 정의를 보편적으로 사용하는데, 키워드에 의해 전달되어야 하고 기본값으로 코딩되지 않은 경우에 필요하다. 키워드용 인자는 * 다음에 코딩되는데, 키워드용 인자가 있고 임의의 위치 인자가 없을 경우 이름 없이 사용될 수 있다.

```python
def f(a, *b, c): print(a, b, c) # 키워드 c 필요
...
f(1, 2, c=3)
1 (2,) 3

def f(a, *, c=None): print(a, c) # 키워드 c 생략 가능
...
f(1)
1 None
f(1, c='spam')
1 spam
```

> 파이썬 3.X 버전부터는 함수를 정의할 때 '키워드 전용 인자(keyword-only arguments)'라는 특별한 인자를 사용할 수 있게 되었어요. 이름 그대로 이 인자들은 함수를 호출할 때 반드시 '키워드'를 사용해서 값을 전달해야만 하는 인자들을 말합니다.

#### 키워드 전용 인자(Keyword-Only Arguments)는 왜 필요할까요?

>보통 함수를 호출할 때는 인자(argument)의 위치만 맞춰서 값을 전달할 수도 있고, 인자 이름(키워드)을 명시해서 값을 전달할 수도 있습니다.

```python
def greet(name, message):
    print(f"Hello, {name}! {message}")

greet("Alice", "How are you?") # 위치 인자 (Alice는 name, How are you?는 message)
greet(message="Good morning", name="Bob") # 키워드 인자 (명확하죠?)
```

>그런데 특정 인자는 **반드시 키워드를 써서 값을 넘겨주도록 강제하고 싶을 때**가 있어요. 왜냐하면:

1.  **가독성 향상:** 함수의 특정 인자가 어떤 의미인지 명확하게 하고 싶을 때. 특히 인자가 많을 때 키워드를 사용하면 훨씬 읽기 쉬워져요.
2.  **버그 예방:** 인자의 순서를 실수로 바꿔서 잘못된 값이 전달되는 것을 막을 수 있어요.

#### 키워드 전용 인자를 어떻게 만들까요?

파이썬에서는 `*` (별표)를 사용해서 키워드 전용 인자를 구분해요.

**1. `*args` 뒤에 오는 인자들**

함수 정의에서 `*` 뒤에 오는 인자들은 모두 키워드 전용 인자가 됩니다. 여기서 `*args`는 파이썬에서 가변 위치 인자를 받을 때 사용하는 표기법인데, 사실 `*`만 단독으로 사용해도 키워드 전용 인자를 만들 수 있어요.

**예시 1: `def f(a, *b, c):`**

```python
def f(a, *b, c): # 여기서 'c'는 키워드 전용 인자입니다.
    print(a, b, c)

# f(1, 2, 3) # 에러 발생! 'c'는 키워드 없이 전달될 수 없습니다.
f(1, 2, c=3) # 올바른 호출. 'c'에 키워드를 사용했습니다.
# 출력: 1 (2,) 3

f(10, c=20) # 'b'는 빈 튜플이 됩니다.
# 출력: 10 () 20
```

*   여기서 `*b`는 `b`가 가변 위치 인자들을 튜플로 묶어서 받는다는 뜻이에요.
*   `c`는 `*b` 뒤에 나왔기 때문에 **반드시 `c=`와 같이 키워드를 써서 값을 전달해야 합니다.**
*   `f(1, 2, c=3)`에서 `1`은 `a`에, `2`는 `*b`에 포함되어 `(2,)` 튜플이 되고, `c=3`은 `c`에 전달됩니다.

**2. `*`만 단독으로 사용한 경우 (가변 위치 인자 없이 키워드 전용 인자만)**

만약 가변 위치 인자(예: `*b`)는 필요 없고, 단순히 특정 인자들만 키워드 전용으로 만들고 싶다면 `*`만 단독으로 사용할 수 있어요.

**예시 2: `def f(a, *, c=None):`**

```python
def f(a, *, c=None): # 여기서 'c'는 키워드 전용 인자입니다.
    print(a, c)

f(1) # 'a'에 1이 전달되고, 'c'는 기본값인 None을 사용합니다.
# 출력: 1 None

f(1, c='spam') # 'a'에 1이 전달되고, 'c'에 'spam'이 전달됩니다. (키워드 사용)
# 출력: 1 spam

# f(1, 'spam') # 에러 발생! 'c'는 키워드 없이 전달될 수 없습니다.
```

*   `def f(a, *, c=None)`에서 `*`는 **"여기까지는 위치 인자고, 이제부터 나오는 인자들은 모두 키워드 전용 인자다"**라는 표시 역할을 해요.
*   `a`는 일반적인 위치 인자이고, `c`는 `*` 뒤에 나왔기 때문에 키워드 전용 인자가 됩니다.
*   `c=None`처럼 기본값을 설정할 수도 있어요. 이렇게 하면 함수를 호출할 때 `c` 값을 생략해도 에러가 나지 않고 기본값이 사용됩니다.

#### 요약하자면:

*   **키워드 전용 인자**는 함수를 호출할 때 **반드시 인자 이름을 명시(`인자이름=값`)해서 값을 전달해야 하는 인자**입니다.
*   함수 정의에서 `*` (별표) 뒤에 오는 인자들이 키워드 전용 인자가 됩니다.
*   `*`만 단독으로 사용하여 가변 위치 인자 없이 키워드 전용 인자만 지정할 수도 있습니다.
*   이를 통해 함수의 가독성을 높이고, 호출 시 발생할 수 있는 오류를 줄일 수 있습니다.

#### 1. `*args` (가변 위치 인자)

>`*b`와 같이 `*` 뒤에 오는 하나의 변수 이름은 **가변 위치 인자**를 의미합니다. 이것은 함수를 호출할 때 몇 개의 위치 인자가 오든지 상관없이 모두 받아서 튜플 형태로 묶어주는 역할을 합니다.

**예시:**

```python
def sum_all(*numbers): # *numbers는 가변 위치 인자
    print(f"받은 숫자들: {numbers}")
    return sum(numbers)

sum_all(1, 2, 3) # 받은 숫자들: (1, 2, 3)
sum_all(10, 20)  # 받은 숫자들: (10, 20)
```

여기서 `numbers`는 `(1, 2, 3)`이나 `(10, 20)`과 같은 **튜플**이 됩니다. 이 인자들을 호출할 때 키워드(`numbers=...`)로 전달하지 않죠? 그냥 위치에 맞춰서 값을 전달하는 일반적인 위치 인자예요.

#### 2. `*` 뒤에 쉼표로 구분되어 오는 인자 (키워드 전용 인자)

>키워드 전용 인자는 **함수 정의에서 `*` (가변 위치 인자) 또는 단독 `*` 뒤에 쉼표(`,`)로 구분되어 나오는 인자**를 말합니다.

**예시 1: `*b` 뒤에 `c`가 오는 경우**

```python
def f(a, *b, c): # 여기서 'b'는 가변 위치 인자, 'c'는 키워드 전용 인자
    print(f"a: {a}, b: {b}, c: {c}")

f(1, 2, 3, c=4)
# 출력: a: 1, b: (2, 3), c: 4
```

*   여기서 `*b`는 `2, 3`을 튜플 `(2, 3)`으로 받아요.
*   `c`는 `*b` 뒤에 **쉼표로 구분되어 나왔기 때문에** 키워드 전용 인자가 됩니다. 따라서 `c=4`처럼 반드시 키워드를 사용해야 합니다.

**예시 2: 단독 `*` 뒤에 `c`가 오는 경우**

```python
def g(a, *, c): # 여기서 'a'는 일반 위치 인자, 'c'는 키워드 전용 인자
    print(f"a: {a}, c: {c}")

g(1, c=2)
# 출력: a: 1, c: 2
```

*   `*`는 "여기까지는 위치 인자고, 지금부터는 키워드 전용 인자다"라는 경계선 역할을 합니다.
*   `c`는 이 단독 `*` 뒤에 **쉼표로 구분되어 나왔기 때문에** 키워드 전용 인자가 됩니다.

#### 핵심 정리:

*   `*변수이름` (예: `*b`)는 **가변 위치 인자**입니다. 호출 시 키워드 없이 위치에 맞춰 전달됩니다.
*   `*` 또는 `*변수이름` 뒤에 **쉼표(`,`)로 구분되어 나오는 인자**들 (예: `c` in `def f(a, *b, c):` 또는 `c` in `def g(a, *, c):`)이 **키워드 전용 인자**입니다. 이들은 호출 시 반드시 키워드(`c=값`)를 사용해야 합니다.

### 파이썬 3.X 함수 어노테이션
> 파이썬 3.X에서는 인자와 반환값이 확장에 사용할 객체값으로 어노테이션되도록 허용하는 함수 정의를 일반적으로 사용한다. 어노테이션은 인자 이름과 기본값 사이에 `:value`로 코딩되며, 인자 리스트 다음에는 `->value`로 코딩된다. 그것들은 함수의 `__annotations__` 속성으로 모이지만, 파이썬 그 자체에서는 특별히 다르게 취급되지 않는다.

```python
def f(a:99, b:'spam'=None) -> float:
    print(a, b)

f(88)
88 None
f.__annotations__
{'a':99, 'b':'spam', 'return': <class 'float'>}
```

### def 문
```python
[decoration]
def name([arg, ... arg=value, ... *arg, **arg]):
    suite
```
> def 문은 새로운 함수나 클래스 메서드를 만든다. 그것은 함수 객체를 생성하고, 생성된 함수 객체를 name 변수에 대입한다. 각각의 함수 객체 호출은 새로운 지역 범위를 생성하며, 대입된 이름은 기본적으로 함수 호출에 대해 지역적인 성격을 띤다(global로 선언되지 않았거나 3.X의 nonlocal이 선언되지 않았다면). 범위에 대한 좀 더 많은 내용은 "이름 공간과 범위 규칙" 섹션 참고.

> 인자는 대입에 의해 전달된다. def 헤더에서 인자는 표14의 인자형식은 함수호출에도 사용될 수 있으며, 표 15처럼 해석된다(함수 호출 문법에 대해서는 "식 문" 섹션 참고)

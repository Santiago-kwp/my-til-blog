## 파이썬 문

> 이번 절에서는 파이썬 문에 관해 설명한다. 각 문장의 문법 형식 목록을 기입하고, 상세한 용법을 알아본다. 복합문으로 구성된 한 개나 여러 개의 다른 문장을 의미하는 문장 형식의 각 묶음은, 헤더 라인 밑에 블록으로 들여쓰게 될 것이다. 한 묶음이 다른 복합문(if, while 등)을 포함하면 그것은 헤더 밑에서 반드시 들여써야 한다. 그렇지 않으면, 그것은 문장 헤더와 같은 라인에 나타날 수 있다. 다음은 둘 다 유효한 생성이다.

```python
if x < 42:
    print(x)
    while x: x = x-1

if x < 42: print(x)
```

> 헤더는 파이썬의 복합문(compound statement) 에서
> 👉 “이제부터 어떤 블록이 시작된다” 라고 알려주는 문장의 앞부분 전체를 말합니다.
> 즉,

- 조건식
- 키워드 (if, while, for, def, class 등)
- 마지막의 콜론 :

까지 포함한 논리적인 문장 단위가 헤더입니다.

### 대입문

```python
target = expression
target1 = target2 = expression
target1, target2 = expression1, expression2
target1 += expression

target1, target2, ... = same-length-iterable
(target1, target2, ...) = same-length-iterable
[target1, target2, ...] = same-length-iterable
target1, *target2, ... = matching-length-iterable
```

> 모든 대입은 대상에 객체 참조를 저장한다. 대입문은 앞에서 제시한 명시적 문법 포맷을 통해 대입을 요청한다.

- 표현식은 객체를 생산한다.
- 대상은 단순히 이름(X)일 수도 있고 제한적 속성(X.attr)일 수도 있으며, 인덱스나 슬라이스(X[i], X[i:j:k])일 수도 있다.
- 대상 내의 변수는 미리 선언되지 않지만, 표현식에서 사용되기 전에 대입돼 있어야 한다

### 파이썬 3.X 키워드용 인자

> 파이썬 3.X는 키워드용 인자를 허용하는 함수 정의를 보편적으로 사용하는데, 키워드에 의해 전달되어야 하고 기본값으로 코딩되지 않은 경우에 필요하다. 키워드용 인자는 \* 다음에 코딩되는데, 키워드용 인자가 있고 임의의 위치 인자가 없을 경우 이름 없이 사용될 수 있다.

```python
def f(a, *b, c): print(a, b, c) # 키워드 c 필요
...
f(1, 2, c=3)
1 (2,) 3

def f(a, *, c=None): print(a, c) # 키워드 c 생략 가능
...
f(1)
1 None
f(1, c='spam')
1 spam
```

> 파이썬 3.X 버전부터는 함수를 정의할 때 '키워드 전용 인자(keyword-only arguments)'라는 특별한 인자를 사용할 수 있게 되었어요. 이름 그대로 이 인자들은 함수를 호출할 때 반드시 '키워드'를 사용해서 값을 전달해야만 하는 인자들을 말합니다.

#### 키워드 전용 인자(Keyword-Only Arguments)는 왜 필요할까요?

> 보통 함수를 호출할 때는 인자(argument)의 위치만 맞춰서 값을 전달할 수도 있고, 인자 이름(키워드)을 명시해서 값을 전달할 수도 있습니다.

```python
def greet(name, message):
    print(f"Hello, {name}! {message}")

greet("Alice", "How are you?") # 위치 인자 (Alice는 name, How are you?는 message)
greet(message="Good morning", name="Bob") # 키워드 인자 (명확하죠?)
```

> 그런데 특정 인자는 **반드시 키워드를 써서 값을 넘겨주도록 강제하고 싶을 때**가 있어요. 왜냐하면:

1.  **가독성 향상:** 함수의 특정 인자가 어떤 의미인지 명확하게 하고 싶을 때. 특히 인자가 많을 때 키워드를 사용하면 훨씬 읽기 쉬워져요.
2.  **버그 예방:** 인자의 순서를 실수로 바꿔서 잘못된 값이 전달되는 것을 막을 수 있어요.

#### 키워드 전용 인자를 어떻게 만들까요?

파이썬에서는 `*` (별표)를 사용해서 키워드 전용 인자를 구분해요.

**1. `*args` 뒤에 오는 인자들**

함수 정의에서 `*` 뒤에 오는 인자들은 모두 키워드 전용 인자가 됩니다. 여기서 `*args`는 파이썬에서 가변 위치 인자를 받을 때 사용하는 표기법인데, 사실 `*`만 단독으로 사용해도 키워드 전용 인자를 만들 수 있어요.

**예시 1: `def f(a, *b, c):`**

```python
def f(a, *b, c): # 여기서 'c'는 키워드 전용 인자입니다.
    print(a, b, c)

# f(1, 2, 3) # 에러 발생! 'c'는 키워드 없이 전달될 수 없습니다.
f(1, 2, c=3) # 올바른 호출. 'c'에 키워드를 사용했습니다.
# 출력: 1 (2,) 3

f(10, c=20) # 'b'는 빈 튜플이 됩니다.
# 출력: 10 () 20
```

- 여기서 `*b`는 `b`가 가변 위치 인자들을 튜플로 묶어서 받는다는 뜻이에요.
- `c`는 `*b` 뒤에 나왔기 때문에 **반드시 `c=`와 같이 키워드를 써서 값을 전달해야 합니다.**
- `f(1, 2, c=3)`에서 `1`은 `a`에, `2`는 `*b`에 포함되어 `(2,)` 튜플이 되고, `c=3`은 `c`에 전달됩니다.

**2. `*`만 단독으로 사용한 경우 (가변 위치 인자 없이 키워드 전용 인자만)**

만약 가변 위치 인자(예: `*b`)는 필요 없고, 단순히 특정 인자들만 키워드 전용으로 만들고 싶다면 `*`만 단독으로 사용할 수 있어요.

**예시 2: `def f(a, *, c=None):`**

```python
def f(a, *, c=None): # 여기서 'c'는 키워드 전용 인자입니다.
    print(a, c)

f(1) # 'a'에 1이 전달되고, 'c'는 기본값인 None을 사용합니다.
# 출력: 1 None

f(1, c='spam') # 'a'에 1이 전달되고, 'c'에 'spam'이 전달됩니다. (키워드 사용)
# 출력: 1 spam

# f(1, 'spam') # 에러 발생! 'c'는 키워드 없이 전달될 수 없습니다.
```

- `def f(a, *, c=None)`에서 `*`는 **"여기까지는 위치 인자고, 이제부터 나오는 인자들은 모두 키워드 전용 인자다"**라는 표시 역할을 해요.
- `a`는 일반적인 위치 인자이고, `c`는 `*` 뒤에 나왔기 때문에 키워드 전용 인자가 됩니다.
- `c=None`처럼 기본값을 설정할 수도 있어요. 이렇게 하면 함수를 호출할 때 `c` 값을 생략해도 에러가 나지 않고 기본값이 사용됩니다.

#### 요약하자면:

- **키워드 전용 인자**는 함수를 호출할 때 **반드시 인자 이름을 명시(`인자이름=값`)해서 값을 전달해야 하는 인자**입니다.
- 함수 정의에서 `*` (별표) 뒤에 오는 인자들이 키워드 전용 인자가 됩니다.
- `*`만 단독으로 사용하여 가변 위치 인자 없이 키워드 전용 인자만 지정할 수도 있습니다.
- 이를 통해 함수의 가독성을 높이고, 호출 시 발생할 수 있는 오류를 줄일 수 있습니다.

#### 1. `*args` (가변 위치 인자)

> `*b`와 같이 `*` 뒤에 오는 하나의 변수 이름은 **가변 위치 인자**를 의미합니다. 이것은 함수를 호출할 때 몇 개의 위치 인자가 오든지 상관없이 모두 받아서 튜플 형태로 묶어주는 역할을 합니다.

**예시:**

```python
def sum_all(*numbers): # *numbers는 가변 위치 인자
    print(f"받은 숫자들: {numbers}")
    return sum(numbers)

sum_all(1, 2, 3) # 받은 숫자들: (1, 2, 3)
sum_all(10, 20)  # 받은 숫자들: (10, 20)
```

여기서 `numbers`는 `(1, 2, 3)`이나 `(10, 20)`과 같은 **튜플**이 됩니다. 이 인자들을 호출할 때 키워드(`numbers=...`)로 전달하지 않죠? 그냥 위치에 맞춰서 값을 전달하는 일반적인 위치 인자예요.

#### 2. `*` 뒤에 쉼표로 구분되어 오는 인자 (키워드 전용 인자)

> 키워드 전용 인자는 **함수 정의에서 `*` (가변 위치 인자) 또는 단독 `*` 뒤에 쉼표(`,`)로 구분되어 나오는 인자**를 말합니다.

**예시 1: `*b` 뒤에 `c`가 오는 경우**

```python
def f(a, *b, c): # 여기서 'b'는 가변 위치 인자, 'c'는 키워드 전용 인자
    print(f"a: {a}, b: {b}, c: {c}")

f(1, 2, 3, c=4)
# 출력: a: 1, b: (2, 3), c: 4
```

- 여기서 `*b`는 `2, 3`을 튜플 `(2, 3)`으로 받아요.
- `c`는 `*b` 뒤에 **쉼표로 구분되어 나왔기 때문에** 키워드 전용 인자가 됩니다. 따라서 `c=4`처럼 반드시 키워드를 사용해야 합니다.

**예시 2: 단독 `*` 뒤에 `c`가 오는 경우**

```python
def g(a, *, c): # 여기서 'a'는 일반 위치 인자, 'c'는 키워드 전용 인자
    print(f"a: {a}, c: {c}")

g(1, c=2)
# 출력: a: 1, c: 2
```

- `*`는 "여기까지는 위치 인자고, 지금부터는 키워드 전용 인자다"라는 경계선 역할을 합니다.
- `c`는 이 단독 `*` 뒤에 **쉼표로 구분되어 나왔기 때문에** 키워드 전용 인자가 됩니다.

#### 핵심 정리:

- `*변수이름` (예: `*b`)는 **가변 위치 인자**입니다. 호출 시 키워드 없이 위치에 맞춰 전달됩니다.
- `*` 또는 `*변수이름` 뒤에 **쉼표(`,`)로 구분되어 나오는 인자**들 (예: `c` in `def f(a, *b, c):` 또는 `c` in `def g(a, *, c):`)이 **키워드 전용 인자**입니다. 이들은 호출 시 반드시 키워드(`c=값`)를 사용해야 합니다.

### 파이썬 3.X 함수 어노테이션

> 파이썬 3.X에서는 인자와 반환값이 확장에 사용할 객체값으로 어노테이션되도록 허용하는 함수 정의를 일반적으로 사용한다. 어노테이션은 인자 이름과 기본값 사이에 `:value`로 코딩되며, 인자 리스트 다음에는 `->value`로 코딩된다. 그것들은 함수의 `__annotations__` 속성으로 모이지만, 파이썬 그 자체에서는 특별히 다르게 취급되지 않는다.

```python
def f(a:99, b:'spam'=None) -> float:
    print(a, b)

f(88)
88 None
f.__annotations__
{'a':99, 'b':'spam', 'return': <class 'float'>}
```

### def 문

```python
[decoration]
def name([arg, ... arg=value, ... *arg, **arg]):
    suite
```

> def 문은 새로운 함수나 클래스 메서드를 만든다. 그것은 함수 객체를 생성하고, 생성된 함수 객체를 name 변수에 대입한다. 각각의 함수 객체 호출은 새로운 지역 범위를 생성하며, 대입된 이름은 기본적으로 함수 호출에 대해 지역적인 성격을 띤다(global로 선언되지 않았거나 3.X의 nonlocal이 선언되지 않았다면). 범위에 대한 좀 더 많은 내용은 "이름 공간과 범위 규칙" 섹션 참고.

> 인자는 대입에 의해 전달된다. def 헤더에서 인자는 표14의 인자형식은 함수호출에도 사용될 수 있으며, 표 15처럼 해석된다(함수 호출 문법에 대해서는 "식 문" 섹션 참고)

#### 표14 정의 시 인자 포맷

| 인자 포맷             | 해석                                           |
| --------------------- | ---------------------------------------------- |
| name                  | 이름이나 위치에 의한 매칭                      |
| name=value            | name이 전달되지 않은 경우의 기본값             |
| \*name                | 새로운 name 튜플로 여분의 위치 인자 모음       |
| \*\*name              | 새로운 name 딕셔너리로 여분의 키워드 인자 모음 |
| \*other, name[=value] | \* 다음의 파이썬 3.X 키워드용 인자             |
| \*, name[=value]      | 앞 라인과 같음(다른 \*가 없을 때)              |

#### 표15 호출 시 인자 포맷

| 인자 포맷      | 해석                              |
| -------------- | --------------------------------- |
| value          | 위치 인자                         |
| name=value     | 키워드(이름에 의한 매칭) 인자     |
| \*iterable     | 시퀀스나 다른 위치 인자 반복 풀기 |
| \*\*dictionary | 키워드 인자 딕셔너리 풀기         |

제공해주신 자료는 파이썬의 `def` 문(함수 정의)과 함수 호출 시 사용되는 인자(argument)의 다양한 형식에 대해 설명하고 있습니다. 이 내용이 다소 어려울 수 있어, 각 부분을 더 자세하고 쉽게 설명해 드릴게요.

### `def` 문: 함수 정의하기

`def` 문은 "define"의 약자로, 새로운 함수를 만들 때 사용합니다. 함수는 특정 작업을 수행하는 코드 블록으로, 필요할 때마다 호출하여 사용할 수 있습니다.

```python
[decoration]
def name([arg, ... arg=value, ... *arg, **arg]):
    suite
```

- **`[decoration]`**: 이 부분은 "데코레이터"라고 불리는데, 함수의 동작을 수정하거나 추가적인 기능을 부여할 때 사용합니다. `@` 기호와 함께 함수 정의 위에 붙습니다. 예를 들어, `@staticmethod`나 `@classmethod` 같은 것들이 있습니다.
  - **예시**:

    ```python
    def my_decorator(func):
        def wrapper():
            print("함수 실행 전!")
            func()
            print("함수 실행 후!")
        return wrapper

    @my_decorator
    def say_hello():
        print("Hello!")

    say_hello()
    # 출력:
    # 함수 실행 전!
    # Hello!
    # 함수 실행 후!
    ```

- **`def`**: 함수 정의를 시작한다는 키워드입니다.

- **`name`**: 함수에 붙일 이름입니다. 이 이름으로 함수를 호출합니다.

- **`([arg, ... arg=value, ... \*arg, **arg])`\*\*: 이 괄호 안에 함수가 받을 인자(매개변수)들을 정의합니다. 인자는 함수가 작업을 수행하는 데 필요한 정보를 외부로부터 받아오는 통로입니다. 다양한 인자 형식이 있으며, 이 부분이 표 14에서 자세히 다루고 있습니다.

- **`:`**: 콜론은 함수 헤더의 끝을 나타내며, 이제부터 함수 본문(suite)이 시작된다는 것을 의미합니다.

- **`suite`**: 함수 본문으로, 함수가 실제로 수행할 코드 블록입니다. 들여쓰기로 구분됩니다.

**핵심 내용:** `def` 문은 함수라는 "설계도"를 만들고, 이 설계도를 `name`이라는 변수에 연결합니다. 함수가 호출될 때마다 새로운 "작업 공간(지역 범위)"이 만들어지며, 이 공간 안에서 변수들이 독립적으로 사용됩니다.

#### 이름 공간과 범위 규칙 (간단 설명)

- **지역 범위 (Local Scope)**: 함수 안에서 정의된 변수들은 기본적으로 그 함수 안에서만 유효합니다. 함수 밖에서는 사용할 수 없습니다.
- **전역 범위 (Global Scope)**: 함수 밖에서 정의된 변수들은 프로그램 전체에서 유효합니다. 함수 안에서도 `global` 키워드를 사용하면 전역 변수를 수정할 수 있습니다.
- **비지역 범위 (Nonlocal Scope, 파이썬 3.X)**: 중첩된 함수(함수 안에 또 다른 함수가 있는 경우)에서 바깥 함수의 변수를 수정하고 싶을 때 `nonlocal` 키워드를 사용합니다.

#### 인자 전달 방식: 대입 (Assignment)

파이썬에서는 함수에 인자를 전달할 때 "대입(assignment)" 방식으로 전달합니다. 즉, 함수를 호출할 때 전달하는 값이 함수 내부의 인자 변수에 할당되는 방식입니다.

#### 표 14: 정의 시 인자 포맷 (함수를 만들 때)

이 표는 `def` 문에서 함수를 정의할 때 사용할 수 있는 다양한 인자(매개변수) 형식을 설명합니다.

| 인자 포맷              | 해석                                             | 예시                      | 설명                                                                                                                                                                                                                                                    |
| :--------------------- | :----------------------------------------------- | :------------------------ | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `name`                 | 이름이나 위치에 의한 매칭                        | `def func(a, b):`         | 가장 기본적인 형태입니다. 함수를 호출할 때 인자의 위치에 따라 값이 매칭됩니다. (예: `func(1, 2)` 면 `a=1`, `b=2`)                                                                                                                                       |
| `name=value`           | name이 전달되지 않은 경우의 기본값               | `def func(a, b=0):`       | 함수를 호출할 때 이 인자에 값을 전달하지 않으면, 미리 지정된 `value`가 기본값으로 사용됩니다. (예: `func(1)` 이면 `a=1, b=0`)                                                                                                                           |
| `*name`                | 새로운 `name` 튜플로 여분의 위치 인자 모음       | `def func(a, *args):`     | 함수를 호출할 때 `a`를 제외하고 남는 모든 "위치 인자"들을 `args`라는 이름의 튜플로 묶어서 받습니다.                                                                                                                                                     |
| `**name`               | 새로운 `name` 딕셔너리로 여분의 키워드 인자 모음 | `def func(a, **kwargs):`  | 함수를 호출할 때 `a`를 제외하고 남는 모든 "키워드 인자"(예: `key=value` 형태)들을 `kwargs`라는 이름의 딕셔너리로 묶어서 받습니다.                                                                                                                       |
| `*other, name[=value]` | `*` 다음의 파이썬 3.X 키워드용 인자              | `def func(a, *, b, c=1):` | 이 형식은 `*` 뒤에 오는 인자들이 반드시 "키워드 인자"로만 전달되어야 함을 강제합니다. 즉, `func(1, 2, 3)`처럼 위치로 전달할 수 없고, `func(1, b=2, c=3)`처럼 `b=`나 `c=`를 명시해야 합니다. 이를 "키워드 전용 인자(keyword-only arguments)"라고 합니다. |
| `*, name[=value]`      | 앞 라인과 같음(다른 `*`가 없을 때)               | `def func(*, b, c=1):`    | 위와 동일합니다. `*` 앞에 위치 인자를 받지 않고, 모든 인자를 키워드 전용으로만 받겠다는 의미입니다.                                                                                                                                                     |

**표 14 예시:**

```python
def example_function(a, b=10, *args, c, d=20, **kwargs):
    print(f"a: {a}")
    print(f"b: {b}")
    print(f"args: {args}") # 튜플로 묶임
    print(f"c: {c}")
    print(f"d: {d}")
    print(f"kwargs: {kwargs}") # 딕셔너리로 묶임

# 호출 예시
example_function(1, 2, 3, 4, c=5, e=6, f=7)
# 출력:
# a: 1
# b: 2
# args: (3, 4)
# c: 5
# d: 20
# kwargs: {'e': 6, 'f': 7}

# 키워드 전용 인자 'c'는 반드시 키워드로 전달해야 합니다.
# example_function(1, 2, 3, 4, 5, e=6, f=7)  # TypeError 발생
```

#### 표 15: 호출 시 인자 포맷 (함수를 사용할 때)

이 표는 `def` 문으로 정의된 함수를 실제로 호출할 때(예: `func(...)`) 인자들을 어떤 형식으로 전달할 수 있는지 설명합니다.

| 인자 포맷      | 해석                              | 예시                                                    | 설명                                                                                                                                |
| :------------- | :-------------------------------- | :------------------------------------------------------ | :---------------------------------------------------------------------------------------------------------------------------------- |
| `value`        | 위치 인자                         | `func(1, 'hello')`                                      | 가장 일반적인 형태입니다. 값이 함수의 매개변수 위치에 따라 순서대로 전달됩니다.                                                     |
| `name=value`   | 키워드(이름에 의한 매칭) 인자     | `func(name='Alice', age=30)`                            | 인자 이름을 명시하여 값을 전달합니다. 위치와 상관없이 이름에 따라 정확하게 매칭됩니다.                                              |
| `*iterable`    | 시퀀스나 다른 위치 인자 반복 풀기 | `my_list = [1, 2, 3]; func(*my_list)`                   | 리스트, 튜플 등 "반복 가능한(iterable)" 객체의 요소들을 개별적인 위치 인자로 풀어서 함수에 전달합니다. (예: `func(1, 2, 3)`과 동일) |
| `**dictionary` | 키워드 인자 딕셔너리 풀기         | `my_dict = {'name': 'Bob', 'age': 25}; func(**my_dict)` | 딕셔너리의 키-값 쌍을 개별적인 키워드 인자로 풀어서 함수에 전달합니다. (예: `func(name='Bob', age=25)`와 동일)                      |

**표 15 예시:**

```python
def greet(name, age, city="Seoul"):
    print(f"Hello, {name}! You are {age} years old and live in {city}.")

# 1. 위치 인자
greet("Alice", 30) # 출력: Hello, Alice! You are 30 years old and live in Seoul.

# 2. 키워드 인자
greet(age=25, name="Bob", city="Busan") # 출력: Hello, Bob! You are 25 years old and live in Busan.
# 순서가 바뀌어도 이름으로 매칭되므로 문제없습니다.

# 3. *iterable (리스트 풀기)
person_data_list = ["Charlie", 40]
greet(*person_data_list) # 출력: Hello, Charlie! You are 40 years old and live in Seoul.
# greet("Charlie", 40) 과 동일

# 4. **dictionary (딕셔너리 풀기)
person_data_dict = {"name": "David", "age": 35, "city": "Jeju"}
greet(**person_data_dict) # 출력: Hello, David! You are 35 years old and live in Jeju.
# greet(name="David", age=35, city="Jeju") 와 동일

# *iterable과 **dictionary 조합
def calculate(a, b, c, op="add"):
    if op == "add":
        return a + b + c
    elif op == "multiply":
        return a * b * c

nums = [1, 2]
options = {"c": 3, "op": "multiply"}
result = calculate(*nums, **options)
print(f"Result: {result}") # 출력: Result: 6 (1 * 2 * 3)
```

### lambda 표현식

> 함수는 lambda 표현식 형식을 통해 생성할 수도 있다. lambda 표현식은 새 함수 객체를 생성하고 이것을 한 이름에 대입하는 대신, 나중에 호출할 수 있도록 반환한다.

`lambda arg, arg, ...: expression`

> lambda에서 각 arg는 def(표 14)와 같고, expression은 추후 호출할 때 나오는 묵시적 반환값이다. expression 내의 코드는 호출될 때까지 실행이 효과적으로 지연된다.

```python
L=lambda a, b=2, *c, **d: [a, b, c, d]
L(1, 2, 3, 4, x=1, y=2)
>>> [1, 2, (3, 4), {'y': 2, 'x': 1}]
```

> lambda는 문장이 아니라 표현식이므로 def를 사용할 수 없는 부분에도 사용할 수 있다(예를 들어, 딕셔너리 리터럴식이나 함수 호출의 인자 리스트 안). lambda는 문장을 실행하는 대신 단일 표현식을 계산하므로, 복잡한 동작에는 어울리지 않는다(복잡한 함수는 def를 사용한다).

제공해주신 자료는 파이썬의 `lambda` 표현식에 대해 설명하고 있습니다. `lambda`는 간단한 함수를 한 줄로 빠르게 만들 때 사용하는 유용한 기능입니다. 이 내용을 더 자세하고 쉽게 설명해 드릴게요.

#### `lambda` 표현식: 간결한 익명 함수

`lambda` 표현식은 `def` 키워드처럼 함수를 정의하지만, 몇 가지 중요한 차이점이 있습니다. 가장 큰 특징은 **익명(anonymous)** 함수를 만든다는 것입니다. 즉, 함수에 이름을 명시적으로 부여하지 않습니다.

```python
lambda arg, arg, ...: expression
```

- **`lambda`**: `lambda` 함수를 정의할 때 사용하는 키워드입니다.
- **`arg, arg, ...`**: 이 부분은 함수가 받을 인자(매개변수)들입니다. `def` 문에서 정의하는 인자 형식(표 14)과 동일하게 `name`, `name=value`, `*name`, `**name` 등을 사용할 수 있습니다. 여러 개의 인자를 콤마(`,`)로 구분합니다.
- **`:`**: 콜론은 인자 목록의 끝과 함수 본문의 시작을 구분합니다.
- **`expression`**: 이 부분은 `lambda` 함수가 실행할 단 하나의 표현식입니다. `lambda` 함수는 이 표현식의 결과를 자동으로 반환합니다. `return` 키워드를 명시적으로 사용할 필요가 없습니다.

**핵심 내용**: `lambda`는 함수 객체를 생성하고, 이 객체를 바로 반환합니다. `def`처럼 이름을 정하고 그 이름에 함수를 대입하는 과정을 거치지 않아도 됩니다. 함수가 정의될 때가 아니라 **호출될 때** 비로소 `expression` 내부의 코드가 실행됩니다.

#### `lambda` 예시 해설

```python
L = lambda a, b=2, *c, **d: [a, b, c, d]
L(1, 2, 3, 4, x=1, y=2)
>>> [1, 2, (3, 4), {'y': 2, 'x': 1}]
```

이 예시를 단계별로 분석해 볼게요.

1.  **`L = lambda a, b=2, \*c, **d: [a, b, c, d]`\*\*:
    - `lambda` 키워드로 익명 함수를 정의합니다.
    - **인자**:
      - `a`: 필수 위치 인자입니다.
      - `b=2`: 기본값이 2인 인자입니다. 제공되지 않으면 2가 사용됩니다.
      - `*c`: `a`, `b`를 제외한 모든 남은 위치 인자들을 `c`라는 튜플로 묶습니다.
      - `**d`: `a`, `b`를 제외한 모든 남은 키워드 인자들을 `d`라는 딕셔너리로 묶습니다.
    - **표현식**: `[a, b, c, d]`는 이 `lambda` 함수가 최종적으로 반환할 리스트입니다.

2.  **`L(1, 2, 3, 4, x=1, y=2)`**:
    - 정의된 `lambda` 함수를 `L`이라는 변수에 대입했으므로, `L`을 함수처럼 호출합니다.
    - **전달된 값들**:
      - `1`: `a`에 할당됩니다. (`a = 1`)
      - `2`: `b`에 할당됩니다. (`b = 2`)
      - `3, 4`: 남은 위치 인자들이므로 `*c`에 의해 `c` 튜플로 묶입니다. (`c = (3, 4)`)
      - `x=1, y=2`: 남은 키워드 인자들이므로 `**d`에 의해 `d` 딕셔너리로 묶입니다. (`d = {'x': 1, 'y': 2}`)
        - 참고: 딕셔너리는 삽입 순서가 보장되지만, 이 예시에서는 튜플이나 리스트처럼 순서가 중요하지 않습니다.

3.  **결과**: `[a, b, c, d]` 표현식이 실행되어 `[1, 2, (3, 4), {'x': 1, 'y': 2}]` 리스트가 반환됩니다. (출력에 `{'y': 2, 'x': 1}`으로 나온 것은 딕셔너리 내부 요소의 순서가 다르게 출력될 수 있기 때문이지만, 본질은 같습니다.)

#### `lambda`의 장점 및 용도

- **간결성**: 한 줄로 함수를 정의할 수 있어 코드가 짧고 깔끔해집니다.
- **익명성**: 이름을 부여할 필요가 없어, 한 번만 사용될 짧은 함수에 적합합니다.
- **표현식 내 사용 가능**: `def` 문은 "문장(statement)"이지만, `lambda`는 "표현식(expression)"입니다. 이 차이가 중요합니다.
  - `def`는 함수 정의 자체를 하나의 문장으로 처리합니다.
  - `lambda`는 값을 생성하는 표현식의 일부로 사용될 수 있습니다.

  **`lambda`를 사용할 수 있는 곳 (예시):**
  1.  **딕셔너리 리터럴 값**:

      ```python
      operation = {
          'add': lambda x, y: x + y,
          'subtract': lambda x, y: x - y
      }
      print(operation['add'](5, 3))    # 8
      print(operation['subtract'](5, 3)) # 2
      ```

  2.  **함수 호출의 인자**: `map()`, `filter()`, `sorted()` 같은 고차 함수(함수를 인자로 받는 함수)에 콜백 함수로 자주 사용됩니다.

      ```python
      # 리스트의 모든 요소에 10 더하기
      numbers = [1, 2, 3, 4]
      doubled_numbers = list(map(lambda x: x + 10, numbers))
      print(doubled_numbers) # [11, 12, 13, 14]

      # 짝수만 필터링하기
      even_numbers = list(filter(lambda x: x % 2 == 0, numbers))
      print(even_numbers) # [2, 4]

      # 튜플 리스트를 두 번째 요소 기준으로 정렬하기
      data = [(1, 'b'), (3, 'a'), (2, 'c')]
      sorted_data = sorted(data, key=lambda item: item[1])
      print(sorted_data) # [(3, 'a'), (1, 'b'), (2, 'c')]
      ```

#### `lambda`의 한계

- **단일 표현식만 가능**: `lambda`는 `expression` 부분에 오직 하나의 표현식만 가질 수 있습니다. 여러 줄의 코드나 `if/else`, `for` 루프와 같은 복잡한 문장들은 `lambda` 안에 직접 넣을 수 없습니다.
  - 만약 복잡한 로직이 필요하다면, 삼항 연산자(ternary operator)를 사용하여 `lambda` 내에서 조건부 표현식을 구현할 수는 있습니다.
  - `lambda x: x * 2 if x > 0 else 0`

- **복잡한 동작에는 부적합**: 위 한계 때문에 복잡하거나 여러 단계를 거치는 함수에는 `def`를 사용하는 것이 훨씬 적합합니다. `def` 함수는 가독성도 좋고 디버깅도 용이합니다.

#### 결론

`lambda` 표현식은 파이썬에서 작고 간단한 함수를 즉석에서 만들 때 매우 유용합니다. 특히 `map`, `filter`, `sorted`와 같은 함수형 프로그래밍 스타일의 함수들과 함께 사용될 때 강력한 시너지를 발휘합니다. 하지만 복잡한 로직이나 여러 줄의 코드가 필요한 경우에는 `def` 키워드를 사용하여 일반 함수를 정의하는 것이 바람직합니다.

### 함수와 메서드 데코레이터

> 파이썬 2.4부터 다음과 같이 함수를 묘사하는 장식 문법을 함수 정의 앞에 놓을 수 있다. 데코레이터로 알려진 이것은 `@`와 함께 코딩되며, 함수적 기법을 위한 명시적 문법을 제공한다. 함수 데코레이터 문법은 다음과 같다.

```python
@decorator
def F():
    ...
```

> 이는 다음의 수동적인 이름 리바인딩과 같다.

```python
def F():
    ...
F = decorator(F)
```

> 이는 특정 함수를 호출 가능한 decorator로 전달한 후, 돌려받은 결과에 다시 함수 이름을 리바인딩하는 효과가 있다. 함수 데코레이터는 함수나 (프락시 객체를 이용한) 추후의 호출을 관리하기 위해 사용된다. 데코레이터는 클래스 내부의 메서드를 포함한 모든 함수 정의에 적용할 수 있다.

```python
class C:
    @decorator
    def M():    # M = decorator(M)과 동일
        ...
```

> 다음의 중첩 장식이 보다 일반적으로 사용된다.

```python
@A
@B
@C
def f(): ...
```

> 데코레이터는 인자 리스트를 가질 수 있다.

```python
@spam(1, 2, 3)
def f(): ...
```

> 이 경우 spam은 함수를 반환하는 함수여야 하며, 그것을 팩터리 함수라 부른다. 이것의 결과는 실제 데코레이터로 사용되며, 필요에 따라 인자 상태를 유지할 수 있다. 데코레이터는 반드시 함수가 정의된 라인의 앞에 나타나야 하며, 같은 라인에 있을 수는 없다(예를 들어, @A def f(): ... 처럼 단일 라인에 쓰는 것은 금지된다)

### return 문

```python
    return [expression]
```

> return 문은 둘러싼 함수에서 빠져나가고 함수를 호출한 결과로 expression 값을 반환한다. expression이 생략되면 기본값인 None이 되며, return을 통하지 않고 빠져나간 함수의 기본 반환값 또한 None이다. 여러 값을 가지는 함수의 결과를 위해서는 튜플을 반환한다.

> 제너레이터 함수에 사용됐을 때 반환의 특별한 의미 체계를 알고 싶다면 아래의 "yield"문을 참고하자.

### yield 문

```python
    yield expression    # 모든 파이썬 버전
    yield from iterable # 3.3과 그 이후 버전
```

> 2.X와 3.X의 yield 표현식은 요청 시 결과를 생성하는 제너레이터 함수를 정의한다. yield를 포함하는 함수는 특별한 방식으로 컴파일되는데, 함수 호출 시, 그것은 제너레이터 객체를 생성하고 반환한다. 이 제너레이터 객체는 반복 컨텍스트에서 결과를 제공하기 위한 반복 프로토콜을 자동으로 지원한다.

> 이는 흔히 식 문으로 코딩되며(예를 들어, 그 자체가 한 라인에), yield는 함수 상태를 일시 정지하고 expression 값을 반환한다. 다음 반복에서 함수의 이전 위치와 변수 상태가 복원되며, 제어는 yield 문 다음에서 바로 재개한다.

> 반복을 끝내거나 함수 끝으로 가기 위해서는 return 문을 사용한다. 3.3 이전에는 제너레이터 함수 return은 반환값을 줄 수 없었다. 그러나 3.3부터는 반환값을 제공할 수 있으며, 이는 예외 객체 속성으로 유지된다.

```python
def generateSquares(N):
    for i in range(N):
        yield i ** 2

G = generateSquares(5)  # __init__, __next__를 가짐
list(G)                 # 지금 결과를 생성
>>> [0, 1, 4, 9, 16]
```

> 표현식으로 사용됐을 때(예를 들어, A = yield X), yield는 제너레이터의 send() 메서드로 전달된 객체를 호출자에 반환하며, 그것이 = 의 오른쪽에 존재하는 단일 아이템이 아닐 경우 괄호로 둘러싸야 한다.(예: A = (yield X) + 42). 이 모드에서는 send(value)를 호출하여 값을 제너레이터로 보내고, 제너레이터가 재개된 후, yield 표현식은 value를 반환한다. 만약 정규 **next()** 메서드나 next() 내장 함수가 사전에 호출되면 yield는 None을 반환한다.

### return vs yield

#### 1. `yield`가 도대체 뭐 하는 건가?

`return`과 `yield`의 차이를 **'공장'** 에 비유해 볼게요.

- **`return` 방식:** 주문이 들어오면 모든 제품(리스트 등)을 한 번에 다 만들어서 상자에 담아 한꺼번에 전달합니다. (만약 100만 개를 주문하면, 100만 개를 다 만들 때까지 창고가 꽉 차고 기다려야 합니다.)
- **`yield` 방식:** 주문이 들어오면 "자, 여기 하나 나왔어!" 하고 **하나씩 던져줍니다.** 그리고는 그 자리에 멈춰서 다음 주문이 올 때까지 기다립니다.

즉, **`yield`는 함수를 "중간에 멈췄다 다시 시작할 수 있게" 만드는 장치**입니다.

### 2. 왜 만들었을까요? (메모리 절약)

가장 큰 이유는 **'메모리 효율'** 때문입니다.

예를 들어 1부터 1,000,000까지의 숫자를 다루는 프로그램을 만든다고 가정해 봅시다.

- **`return`을 쓰면:** 1부터 1,000,000까지의 숫자가 담긴 거대한 리스트를 메모리에 한꺼번에 올려야 합니다. 컴퓨터가 힘들어하겠죠?
- **`yield`를 쓰면:** 딱 필요한 그 순간에 숫자 하나만 메모리에 올립니다. 1을 쓰고 나면 1은 메모리에서 사라지고 2가 그 자리를 차지합니다. 전체를 한꺼번에 들고 있을 필요가 없으니 훨씬 가볍고 빠릅니다.

#### 3. 어떤 때 쓰나요?

1.  **데이터가 너무 많을 때:** 로그 파일 읽기, 대용량 데이터베이스 조회 등 한꺼번에 메모리에 올리면 컴퓨터가 뻗어버릴 것 같은 작업을 할 때 씁니다.
2.  **무한한 데이터를 다룰 때:** "0부터 무한대까지 숫자를 하나씩 출력해" 같은 코드를 만들고 싶을 때, `return`으로는 불가능하지만 `yield`로는 가능합니다.
3.  **반복문(for 문)이 필요할 때:** `for i in range()`처럼 값을 순서대로 하나씩 꺼내 쓰는 것이 자연스러운 상황에서 매우 유용합니다.

---

#### 코드 다시 보기 (쉽게 풀이)

```python
def generateSquares(N):
    for i in range(N):
        yield i ** 2  # 여기서 '멈춤'하고 값을 밖으로 던짐

G = generateSquares(5)  # 이때 함수는 실행되지 않고 '준비 상태'가 됨
```

1.  `G = generateSquares(5)`를 해도 함수 내부 코드가 바로 다 실행되지 않습니다. 그냥 **"언제든 꺼낼 수 있는 준비물(제너레이터 객체)"** 만 하나 챙긴 겁니다.
2.  `list(G)`나 `next(G)`를 호출하면 비로소 함수가 첫 번째 `yield`까지 실행해서 값을 줍니다.
3.  다시 호출하면, **지난번에 멈췄던 그 줄 바로 다음부터** 다시 실행을 시작합니다.
4.  함수가 끝날 때까지 이 과정을 반복합니다.

#### 요약

- **`return`**: "이거 다 가져가!" (결과물 전체 반환)
- **`yield`**: "이거 하나 가져가. 다음 거 필요하면 다시 말해!" (필요할 때 하나씩 생성)

이것을 전문 용어로 **'지연 평가(Lazy Evaluation)'**라고 합니다. "필요할 때까지 계산을 미룬다"는 뜻

#### list(G)의 내부 작동 단계 (일종의 '가이드 모드')

> 파이썬의 list() 함수는 인자로 들어온 것이 '반복 가능한 객체(제너레이터 등)'라면, 그 안에 값이 없을 때까지 **"다음 값 줘!" (**next**)**를 끊임없이 반복합니다.

`list(G)`를 호출하는 순간, 파이썬 내부에서는 **"아, 이 녀석이 안에 든 데이터를 전부 다 달라고 하는구나!"**라고 판단하고, `G`가 바닥날 때까지 함수를 계속해서 툭툭 건드리는 과정을 거칩니다.

구체적인 작동 순서는 다음과 같습니다.

### `list(G)`의 내부 작동 단계 (일종의 '가이드 모드')

파이썬의 `list()` 함수는 인자로 들어온 것이 '반복 가능한 객체(제너레이터 등)'라면, 그 안에 값이 없을 때까지 **"다음 값 줘!" (`__next__`)**를 끊임없이 반복합니다.

1.  **준비:** `list()`는 빈 리스트 `[]`를 메모리에 만듭니다.
2.  **첫 번째 요청:** `list()`가 `G`에게 "첫 번째 값 줘!"라고 합니다.
    - 함수(`generateSquares`)가 처음부터 실행되어 `i=0`일 때 `yield 0**2`를 만납니다.
    - 함수는 `0`을 밖으로 던지고 **그 자리에 꽁꽁 얼어붙듯 멈춥니다.** (변수 `i` 값인 0과 `for`문의 상태를 기억한 채로요!)
    - `list()`는 전달받은 `0`을 빈 리스트에 넣습니다: `[0]`
3.  **두 번째 요청:** `list()`가 다시 "다음 값 줘!"라고 합니다.
    - 함수는 멈췄던 지점(`yield` 바로 다음)에서 바로 깨어납니다.
    - `for`문의 다음 순서로 넘어가 `i=1`이 됩니다. `yield 1**2`를 만나 `1`을 던지고 다시 **멈춥니다.**
    - `list()`는 리스트에 추가합니다: `[0, 1]`
4.  **반복:** 이 과정을 `i=4`가 될 때까지 똑같이 반복합니다.
5.  **종료 (중요!):** `i=4`까지 다 돌고 나면 `for`문이 끝납니다. 이때 함수는 더 이상 `yield`를 만날 수 없게 되는데, 파이썬은 이럴 때 **`StopIteration`**이라는 신호를 보냅니다.
6.  **마무리:** `list()`는 이 신호를 받는 순간 "아, 이제 끝났구나!"하고 그동안 모았던 `[0, 1, 4, 9, 16]`을 반환하고 작업을 종료합니다.

---

#### 왜 이렇게 번거롭게 할까요?

질문하신 내용처럼 **결국 리스트로 만들 거면 `return`으로 한 번에 만드는 게 낫지 않나?** 싶으실 겁니다. 하지만 `yield`의 진가는 **리스트로 만들지 않을 때** 나옵니다.

만약 `for x in G:` 같은 반복문을 쓴다면:

```python
for x in G:
    print(x)
```

이 코드는 **메모리에 리스트 전체를 올리지 않습니다.**

1. `0`을 하나 받아서 `print`하고, 그 `0`은 메모리에서 버립니다.
2. 다음 `1`을 받아서 `print`하고, 그 `1`은 메모리에서 버립니다.
3. ...

즉, **데이터가 1억 개여도 메모리는 딱 '숫자 하나' 크기만큼만 사용**하게 됩니다.

**결론:**

- `list(G)`를 하면 함수 내부가 끝까지 다 실행되면서 리스트로 변환되지만,
- `for` 문이나 `next()`로 하나씩 꺼내 쓰면 **메모리를 극도로 아끼면서도 거대한 데이터를 순차적으로 처리**할 수 있습니다. 이것이 `yield`를 쓰는 가장 큰 이유입니다.

### import 문

> import 문은 모듈 접근 기능을 제공하며, 모듈을 전체적으로 임포트한다. 여러 모듈  중 각 모듈은 모듈명.속성명로 가져온 이름을 포함하낟. 파이썬 파일 최상위의 대입은 모듈 객체 속성을 생성한다. 선택적 as 절은 변수 name을 임포트된 모듈 객체에 대입하고 원래 module 이름을 제거하는데, 이는 긴 모듈 이름이나 패키지 경로를 위한 짧은 동의어를 제공할 때 유용하다. 선택적 package 접두사는 패키지 디렉토리 경로를 표시한다.

```python
import [package.]* module [as name]
    [, [package.]* module [as name]]*
```


> module은 대상 모듈을 명명하며, 대개 파이썬 소스 코드나 컴파일된 바이트 코드 파일이다. module은 파일명 확장을 빼고 제공하며, package 경로에 중첩되어 있지 않다면 일반적으로 모듈 검색 경로 디렉토리에 위치해야 한다.

> 프로그램에 의해 모듈이 처음 임포트될 때, 그 소스 코드 파일은 필요할 경우 바이트 코드로 컴파일되며(그리고 가능하면 .pyc 파일로 저장), 대입을 통해 모듈 객체 속성을 생성하기 위해 처음부터 끝까지 수행된다. 파이썬 2.X와 3.1 그리고 그 이전 버전에서, 바이트 코드 파일은 같은 기본명(예를 들어, module.pyc)으로 소스 코드 파일이 있는 디렉터리에 저장된다. 파이썬 3.2 이후부터는 바이트 코드는 소스 코드 파일 디렉터리의 __pycache__라는 하위 디렉터리에 버전을 식별할 수 있는 기본명(예: module.cpython-33.pyc)으로 저장된다.

#### 1. 먼저 큰 그림: 패키지와 모듈이란?

#### ✅ 모듈(module)

*   **모듈은 하나의 파이썬 파일(.py)** 입니다.
*   변수, 함수, 클래스 등을 담은 **코드 묶음**이라고 생각하면 됩니다.

```text
math.py
my_utils.py
calculator.py
```

➡ 이런 파일 각각이 **모듈**

```python
# my_utils.py
def add(a, b):
    return a + b
```

***

#### ✅ 패키지(package)

*   **여러 모듈을 폴더로 묶은 것**
*   보통 폴더 안에 여러 `.py` 파일이 있고, (전통적으로는) `__init__.py` 파일이 있음

```text
mypackage/
 ├─ __init__.py
 ├─ module1.py
 └─ module2.py
```

➡ `mypackage`가 **패키지**,  
➡ `module1`, `module2`가 **모듈**

***

#### 2. import 문 형식 해설

#### 원문

```python
import [package.]* module [as name]
    [, [package.]* module [as name]]*
```

#### 구성 요소 하나씩 분해하면:

| 요소         | 의미                   |
| ---------- | -------------------- |
| `package.` | 패키지 이름 (여러 단계 가능)    |
| `module`   | 실제 불러올 모듈            |
| `as name`  | 모듈 이름에 별명(alias)을 붙임 |
| `,`        | 여러 모듈을 한 줄에서 임포트 가능  |

#### 3. “import 문은 모듈 접근 기능을 제공하며, 모듈을 전체적으로 임포트한다” 의미

#### ✅ 핵심 의미

```python
import math
```

*   `math` **모듈 전체**를 메모리로 불러옴
*   모듈 안의 내용은 **math.이름** 형태로 접근

```python
import math

print(math.pi)
print(math.sqrt(16))
```

❌ 이렇게는 접근 불가:

```python
print(pi)       # NameError
print(sqrt(4))  # NameError
```

👉 **모듈 전체를 그대로 가져오므로 접두사 `math.`가 필요**

***

#### 4. “각 모듈은 모듈명.속성명으로 가져온 이름을 포함한다” 설명

#### 이 문장의 의미

*   모듈 안에 정의된 모든 것(변수, 함수, 클래스)은  
    **모듈 객체의 속성(attribute)** 이 됨

```python
# mymod.py
x = 10

def func():
    print("hello")
```

```python
import mymod

print(mymod.x)      # 10
mymod.func()        # hello
```

➡ `x`, `func`는 **mymod 객체의 속성**
➡ `mymod.x`, `mymod.func()` 형태로 접근

***

#### 5. “파이썬 파일 최상위의 대입은 모듈 객체 속성을 생성한다”

#### 쉽게 말하면:

> **모듈 파일(.py) 안에서 맨 위 레벨에서 선언한 것은 전부 모듈의 속성이 된다**

```python
# example.py
a = 100          # 최상위 대입
b = "hello"

def foo():
    pass
```

```python
import example

print(example.a)    # 100
print(example.b)    # hello
example.foo()
```

✅ `a`, `b`, `foo` → 전부 `example` 모듈의 속성

❌ 함수 안에서 만든 변수는 속성이 아님

```python
def bar():
    c = 10

# example.c 는 존재하지 않음
```

#### 6. “as 절은 긴 모듈 이름이나 패키지 경로를 위한 짧은 동의어”

#### 기본 예제

```python
import numpy
```

```python
numpy.array([1, 2, 3])
```

👇 보통 이렇게 씀:

```python
import numpy as np
```

```python
np.array([1, 2, 3])
```

### assert 문

```python
assert expression [, message]
```

> assert 문은 디버깅 확인을 위해 수행한다. 만약 expression이 거짓이면 AssertionError를 발생시키고, message가 제공됐다면 그것을 생성자 인자로 전달한다. -0 명령 라인 플래그는 assert 문을 제거한다(포함되지도 않고 실행되지도 않는다)

***

#### 1. assert 문 기본 형태

```python
assert expression [, message]
```

구성 요소부터 정리해보면:

| 요소           | 의미                          |
| ------------ | --------------------------- |
| `assert`     | “이 조건이 반드시 참이어야 한다”고 선언     |
| `expression` | 검사할 조건식 (True / False로 평가됨) |
| `message`    | 조건이 거짓일 때 함께 출력할 설명(선택)     |

***

#### 2. assert 문의 핵심 개념 한 문장 요약

> **assert는 “이 지점에 도달했을 때 반드시 참이어야 하는 조건”을 코드로 명시하는 디버깅용 검사 장치이다.**

***

#### 3. “assert 문은 디버깅 확인을 위해 수행한다” 설명

#### ✓ 의미

*   `assert`는 **프로그램의 정상 동작을 보장하기 위한 가정(assumption)** 을 코드에 적어두는 것
*   주 목적은 **버그를 빨리 발견**하는 것
*   일반적인 입력 검증이나 예외 처리용이 아님

#### 예제

```python
def divide(a, b):
    assert b != 0
    return a / b
```

*   개발자는 “이 함수에 들어올 때 b는 0이 아니어야 한다”라고 **가정**
*   가정이 깨지면 즉시 오류로 종료 → 문제 위치를 바로 알 수 있음

***

#### 4. “expression이 거짓이면 AssertionError를 발생시킨다”

#### 동작 방식

```python
assert expression
```

은 내부적으로 다음과 같은 의미입니다:

```python
if not expression:
    raise AssertionError
```

#### 예제

```python
x = -1
assert x > 0
```

실행 결과:

```text
AssertionError
```

#### True일 경우

```python
assert 3 > 1
print("정상 실행")
```

출력:

    정상 실행

➡ 조건이 참이면 **아무 일도 일어나지 않고 그냥 통과**

***

#### 5. “message가 제공됐다면 그것을 생성자 인자로 전달한다”

#### 이 말이 뜻하는 것

```python

```

는 내부적으로 다음과 같습니다:

```python

    raise AssertionError(메시지)
```

#### 예제

```python

```

출력:


➡ 메시지는 **오류의 원인을 설명하는 디버깅 힌트**

***

#### 6. AssertionError는 어떤 오류인가?

*   파이썬에 **기본 내장된 예외 클래스**
*   개발자의 가정이 깨졌을 때 사용

```python

```

출력:

### with 문

> 컨텍스트 매니저(context manager)를 사용하여 자원을 생성(획득) -> 사용 -> 자동 정리(반납)을 보장하는 문법

> 가장 대표적인 자원이 바로:
- 파일
- 락(lock)
- 네트워크 연결
- 데이터베이스 커서

```python
with expression [as variable]:  # 3.0/2.6. + 
    suite

with expression [as variable]
        [, expression [as variable]]*:  # 3.1/2.7. +
    suite
```

- expression: 컨텍스트 매니저 객체를 반환해야 함
    - 예: `open()`
- as variable: 그 객체를 받을 변수


> 자원을 획득하고, 사용하고, 반납할 때 주로 사용한다. 

```python
f = open('myFile.txt', 'w', encoding='utf8')
f.write("test")
f.close()
```

> 다음과 같이 좀 더 간결하고, 안전하게 사용할 수 있다.

```python
with open('mytextfile.txt', 'w', encoding='utf8') as f:
    f.write("test")
```


> with 문은 컨텍스트 관리자 코드의 중첩된 블록을 감싸며, 블록 엔트리 작업을 실행할 수 있다. 그리고 예외 발생 여부와 상관없이 그 블록 종료 작업이 실행되도록 보장한다. with는 컨텍스트 관리자를 가진 객체에 한해 종료 작업에 대한 try/finally의 대안이 될 수 있다. 

> expression은 컨텍스트 관리자 프로토콜을 지원하는 객체를 반환한다고 가정한다. 이 객체는 생략 가능한 as 절이 존재할 경우, variable에 대입될 값 또한 반환할 수 있다. 클래스들은 컨텍스트 관리자를 커스터마이징해 정의할 수 있으며, 파일 및 스레드 같은 일부 내장 자료형은 파일을 닫거나 스레드 잠금을 푸는 등의 종료 작업을 할 수 있는 컨텍스트 관리자를 제공한다.

### 컨텍스트 관리자

#### 1. 컨텍스트 관리자(Context Manager)란?

#### 한 문장 정의

> **컨텍스트 관리자란 “특정 코드 블록의 시작과 끝에서 자동으로 수행되어야 할 동작을 정의한 객체”이다.**

주로 다루는 대상은 **자원(resource)** 입니다.

#### 자원의 예

*   파일 (열기 → 닫기)
*   락(lock) (획득 → 해제)
*   데이터베이스 연결 (연결 → 종료)
*   트랜잭션 (시작 → 커밋/롤백)

➡ 공통점:  
**사용 전 준비 작업 + 사용 후 정리 작업이 반드시 필요**

***

#### 2. 왜 컨텍스트 관리자가 필요한가?

#### 기존 방식: try / finally

```python
f = open("data.txt", "w")
try:
    f.write("hello")
finally:
    f.close()
```

*   ✅ 예외가 발생해도 `close()` 실행 보장
*   ❌ 코드가 장황하고 반복적
*   ❌ 중첩되면 가독성 급락

***

#### with 문 사용

```python
with open("data.txt", "w") as f:
    f.write("hello")
```

*   ✅ 짧음
*   ✅ 안전함
*   ✅ 예외 발생 여부와 관계없이 정리 작업 수행

➡ `with` 문은 **try/finally의 구조적 대체재**

***

#### 3. 설명 문장 1 해설

> **with 문은 컨텍스트 관리자 코드의 중첩된 블록을 감싸며,  
> 블록 엔트리 작업을 실행할 수 있다.  
> 그리고 예외 발생 여부와 상관없이 그 블록 종료 작업이 실행되도록 보장한다.**

#### 이 말을 풀면

*   `with` 블록에 들어갈 때  
    → **엔트리(entry) 작업 실행**
*   `with` 블록을 나갈 때  
    → **종료(exit) 작업 자동 실행**
*   중간에 예외가 발생해도  
    → **종료 작업은 반드시 실행**

즉,

```text
enter → (사용) → exit
```

이 구조를 **언어 차원에서 강제**해주는 것이 `with`

***

#### 4. 컨텍스트 관리자 프로토콜이란?

#### 핵심 정의

> **컨텍스트 관리자 프로토콜은  
> `__enter__()`와 `__exit__()` 두 메서드를 구현하는 규약이다.**

이 두 메서드를 가진 객체를  
➡ **컨텍스트 관리자**라고 부를 수 있습니다.

***

#### 필수 메서드

```python
__enter__(self)
__exit__(self, exc_type, exc_value, traceback)
```

***

#### 5. with 문의 내부 동작 순서 (중요)

다음 코드를 예로 들면:

```python
with expression as variable:
    suite
```

#### 실제 내부 동작은 개념적으로 다음과 같습니다:

```python
manager = expression
value = manager.__enter__()
try:
    variable = value
    suite
finally:
    manager.__exit__(예외정보)
```

***

#### 단계별 설명

1️⃣ `expression` 평가  
→ 컨텍스트 관리자 객체 생성

2️⃣ `__enter__()` 호출  
→ 반환값이 있다면 `as variable`에 대입

3️⃣ 블록 수행 (`suite`)

4️⃣ **항상** `__exit__()` 호출

*   예외 없으면 → `exc_type = None`
*   예외 있으면 → 예외 정보 전달

***

#### 6. 설명 문장 2 해설

> **expression은 컨텍스트 관리자 프로토콜을 지원하는 객체를 반환한다고 가정한다.**

즉:

*   `with` 문은 **아무 객체나 받을 수 없음**
*   반드시 **`__enter__`와 `__exit__`를 가진 객체**

```python
# 가능
with open(...) as f:

# 불가능
with 123:
```

***

> **이 객체는 생략 가능한 as 절이 존재할 경우, variable에 대입될 값 또한 반환할 수 있다.**

#### 의미

*   `__enter__()`의 반환값이  
    → `as 변수`로 들어감

```python
with open("a.txt") as f:
    pass
```

➡ `f`는 `open()`이 아니라  
➡ `open().__enter__()`의 반환값

***

#### 7. 내장 컨텍스트 관리자의 예

#### ✅ 파일 객체

```python
with open("file.txt") as f:
    pass
```

*   `__enter__()` : 파일 열기
*   `__exit__()` : 파일 닫기

***

#### ✅ 락(lock)

```python
with lock:
    critical_section()
```

*   `__enter__()` : 락 획득
*   `__exit__()` : 락 해제

***

#### ✅ 데이터베이스 트랜잭션

```python
with conn:
    cursor.execute(...)
```

*   정상 종료 → commit
*   예외 발생 → rollback

***

#### 8. 커스텀 컨텍스트 관리자 만들기

```python
class MyContext:
    def __enter__(self):
        print("enter")
        return self

    def __exit__(self, exc_type, exc_value, tb):
        print("exit")
```

```python
with MyContext() as ctx:
    print("inside")
```

출력:

    enter
    inside
    exit

***

#### 9. 예외와 **exit**

```python
def __exit__(self, exc_type, exc_value, traceback):
    return True  # 예외를 삼킴
```

*   `True` 반환 → 예외 무시
*   `False` / None → 예외 전파

이 점 때문에 컨텍스트 관리자는  
**정리 + 예외 처리 제어**까지 가능

***

#### 10. 설명 문장 전체 요약 번역

> ✅ with 문은  
> **코드 블록의 시작과 종료 시점을 기준으로  
> 자동으로 실행할 작업을 정의할 수 있게 해주며,  
> 예외가 발생해도 종료 작업을 보장한다.**

> ✅ 이를 위해 with는  
> **`__enter__` / `__exit__`라는 프로토콜을 구현한 객체만을 사용한다.**

> ✅ 파일, 락, 스레드 등은  
> 이미 이러한 컨텍스트 관리자를 제공한다.

## ✅ 최종 핵심 요약

*   **컨텍스트 관리자**:  
    “블록 전후 작업을 자동화한 객체”
*   **컨텍스트 관리자 프로토콜**:  
    `__enter__`, `__exit__` 메서드 규약
*   **with 문**:  
    try/finally를 문법 수준에서 안전하게 추상화한 구조

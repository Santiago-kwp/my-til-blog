## 내장 자료형과 연산자

### 연산자와 우선순위
> 표1은 파이썬의 표현식 연산자를 보여준다. 연산자들이 괄호 없이 혼합되어 사용된 식에서는 표 안에서 아래에 있는 연산자들이 위에 있는 연산자들보다 연산의 우선순위를 가진다.

#### 원자적 용어와 동적 타이핑
> 표 1에서 대체 가능한 표현식 항목인 X, Y, Z, i, j, k는 다음 중 하나일 수 있다.
- 가장 최근에 대입된 값으로 대체되는 변수명
- "파이썬 내장 자료형" 섹션에 정의된 리터럴식
- 이 표의 아무 행에서나 볼 수 있는 중첩식

✏️ 원자적 용어란, 더 이상 연산자로 쪼갤 수 없는 “표현식의 최소 단위”를 말한다.
1. 변수명 (이름, name)
```
x
total
my_value
```
- 가장 최근에 대입된 값을 참조
- 자체로 하나의 완전한 표현식
- 더 쪼갤 수 없음

2. 리터럴(literal)
```
10
3.14
"hello"
True
None
```
- 이미 값 자체
- 연산자를 포함하지 않음

3. 괄호로 묶인 표현식
```
(x + y)
```
- 내부는 복합 표현식이지만
- 겉에서 보면 하나의 단위로 취급

4. 속성 참조, 인덱싱, 호출 등 (문법적으로 원자 취급)
표 1의 아래쪽에 있는 것들입니다:
```python
X[i]        # 인덱싱
X[i:j:k]    # 슬라이싱
X(args)     # 함수 호출
X.attr      # 속성 참조
```
- 더 높은 우선순위
- 문법적으로 “붙어서 하나처럼” 작동


> 파이썬 변수는 동적 타이핑 모델을 따른다. 그것들은 미리 선언되지 않으며, 대입될 때 생성된다. 그리고 값을 가리키는 객체 참조를 가지는데, 어떠한 형태의 객체라도 참조할 수 있다. 또한 기본값을 가지지 않으므로 표현식에 나타나기 전에 반드시 미리 대입돼야(assigned) 한다. 변수명의 대소문자는 항상 구분된다. 변수에 의해 참조된 객체는 자동으로 생성되고, 그것이 더는 사용되지 않을 때 CPython의 참조 카운터를 사용하는 파이썬 가비지 컬렉터에 의해 자동으로 회수된다.

> 표 1에서 대체 가능한 attr은 반드시 한 속성의 리터럴명이어야 한다. args1은 "def문" 섹션에 정의된 대로 형식 인자 리스트이다. args2는 "식 문" 섹션에 정의된 대로 입력 인자 리스트이다. 그리고 리터럴은 3.X에 한해 원자적 표현식으로 사용된다.

> 컴프리헨션의 문법과 표 1 중 마지막 세 행의 데이터 구조 리터럴들(튜플, 리스트, 딕셔너리, 집합)은 "파이썬 내장 자료형" 절을 참고하자

#### 원자적 용어 vs primary

> **원자적 용어(atom)** 는 *표현식의 최소 구성 요소*이고,  
> **primary** 는 *그 atom 위에 인덱싱·속성 접근·호출 등을 덧붙여 만든 표현식*이다.

즉, **atom ⊂ primary**

1. 파이썬 문법에서의 공식 정의

파이썬 레퍼런스 문법에는 다음과 같은 구조가 있습니다:

```text
atom        → NAME | NUMBER | STRING | ... | (expression) | ...
primary     → atom
            | primary "[" slicing "]"
            | primary "." NAME
            | primary "(" [arguments] ")"
            | primary "[" list_display "]"
```

이걸 말로 풀면:

*   **atom**: 더 나눌 수 없는 가장 기본적인 표현식
*   **primary**: atom에 **후행 연산(postfix)** 을 0개 이상 붙인 결과

***

2. atom(원자적 용어)이란?

정의

**atom**은 파이썬 표현식 문법에서  
👉 *더 이상 문법적으로 쪼갤 수 없는 최소 단위*입니다.

atom의 예

```python
x
42
"hello"
True
None
(a + b)
[1, 2, 3]
{1, 2}
```

특징:

*   자체로 완전한 표현식
*   연산자 없이 값이 확정
*   식의 “출발점”

✅ 이 단계에서는 **호출, 인덱싱, 속성 접근이 아직 없음**

***

3. primary란?

정의

**primary**는 atom을 기반으로 해서  
👉 **인덱싱, 슬라이싱, 속성 접근, 호출** 같은 연산을 *연속적으로 적용할 수 있는 표현식*입니다.

primary의 예

```python
x
x[0]
x[0][1]
obj.attr
obj.method()
f(1)(2).attr[0]
```

이 중에서:

*   `x` → atom이자 primary
*   `x[0]` → **atom은 아님**, primary
*   `obj.method()` → primary
*   `f(1)(2).attr[0]` → 계속 확장된 primary

***

4. 핵심 차이 포인트 정리

| 구분     | atom (원자적 용어)     | primary                 |
| ------ | ----------------- | ----------------------- |
| 문법적 위치 | 가장 바닥             | atom **위 단계**           |
| 역할     | 표현식의 최소 단위        | 실제 연산이 붙는 대상            |
| 후행 연산  | ❌ 불가              | ✅ 가능                    |
| 예시     | `x`, `10`, `"hi"` | `x[i]`, `x.attr`, `x()` |
| 중첩 가능성 | ❌ 자체는 단순          | ✅ 무한히 체인 가능             |

***

5. 왜 이런 구분이 필요한가?

예제 1

```python
obj.method()[0]
```

문법적으로 보면:

1.  `obj` → atom
2.  `obj.method` → primary
3.  `obj.method()` → primary
4.  `obj.method()[0]` → primary

👉 atom은 **하나**, primary는 **계속 확장**됨

***

예제 2

```python
(x + y)[0]
```

*   `(x + y)` → atom (괄호 덕분에 하나의 atom)
*   `(x + y)[0]` → primary

👉 그래서 **괄호가 중요한 이유**도 여기서 나옵니다.

***


*   많은 입문서에서
    *   atom, primary, expression
    *   모두 그냥 “표현식”이라고 뭉뚱그려 설명함
*   그래서 primary까지도 “원자적”이라고 느껴지는 경우가 생김

하지만 **엄밀히 말하면**:

*   `x[0]`, `f()` 는 **원자적 아님**
*   대신 **primary**

***

### 연산자 종류
> 일반적으로 모든 내장 자료형은 표 2에 있는 비교 및 불린(Boolean) 연산을 지원한다(단, 파이썬 3.X는 딕셔너리 및 혼합 비수치 자료형에 대한 크기 비교를 지원하지 않는다).

> 불린 True는 0 이외의 숫자나 비지 않은 컬렉션 객체(리스트, 딕셔너리 등), 불린 값을 가진 모든 객체를 의미한다. 내장명 True와 False는 참과 거짓 값으로 미리 대입되며, 사용자 지정 표시의 정수 1과 0처럼 동작한다. 특별한 객체인 None은 거짓이며 파이썬 내에서 자주 등장한다.

> 비교는 True나 False를 반환하며 결과를 결정하기 위해 필요에 따라 자동으로 복합 객체에 재귀적으로 적용된다.

> 불린 and와 or 연산자는 결과가 도출되는 즉시 멈추며, 결과를 나타내는 연산자 좌측이나 우측 피연산자 객체의 불린 값을 반환한다.

#### 표2 비교와 불린 연산

| 연산자     | 설명     |
| ------ | ----------------- |
| X < Y | 명확히 작은 |
| X <= Y | 작거나 같은 |
| X > Y | 명확히 큰 |
| X >= Y | 크거나 같은 |
| X == Y | 등가인 |
| X != Y | 등가가 아닌(파이썬 2.X에 한해 X<>Y와 동일) |
| X is Y | 같은 객체|
| X is not Y | 객체 일치 부정|
| X < Y < Z| 연쇄 비교|
| not X | X가 거짓이면 True, 아니면 False|
| X or Y | X가 거짓이면 Y, 아니면 X|
| X and Y | X가 거짓이면 X, 아니면 Y|

#### 표3 시퀀스 연산 (문자열, 리스트, 튜플, bytes, bytearray)
| 연산자     | 설명     | 클래스 메서드 |
| ------ | ----------------- | ---------|
|X in S, X not in S|멤버십 테스트| `__contains__`, `__iter__`, `__getitem__`|
|S1 + S2|연결| `__add__`|
|S * N, N * S|반복|`__mul__`|
|S[i]|오프셋에 의한 인덱스|`__getitem__`|
|S[i:j], S[i:j:k]|슬라이싱: 오프셋 i부터 j-1까지 S안에 있는 아이템, 간격 k는 생략 가능|`__getitem__`|

### 시퀀스 연산

#### 인덱싱: `S[i]`
- 오프셋에 있는 컴포넌트를 가져온다(첫 아이템은 오프셋 0에)
- 음의 인덱스는 끝에서부터 거꾸로 센다(마지막 아이템은 오프셋 -1에)
- `S[-2]`는 끝에서 두 번째 아이템을 가져온다

#### 슬라이싱: `S[i:j]`
- i부터 j-1까지 시퀀스의 인접한 부분을 추출한다.
- S[1:3]은 오프셋 1부터 3까지 가져온다. 그러나 3을 포함하지는 않는다.
- S[1:]은 오프셋 1부터 끝까지(len(S)-1) 가져온다.
- S[:-1]은 오프셋 0부터 마지막 아이템까지 가져온다. 그러나 마지막 아이템은 포함하지는 않는다.
- S[:]는 시퀀스 객체 S의 최상위(얕은) 복사본을 만든다.

#### 확장 슬라이싱: `S[i:j:k]`
- 세 번째 아이템 k는 1을 기본값으로 갖는 간격이며, 각 추출 아이템의 오프셋에 더해진다.
- S[::2]는 전체 시퀀스 S의 홀수 번째 아이템들이다.
- S[::-1]은 거꾸로 된 시퀀스 S다.
- S[4:1:-1]은 오프셋 4부터 1까지 거꾸로 된 결과를 가져온다. 그러나 1을 포함하지는 않는다.

#### 슬라이스 대입: `S[i:j:k]`
- 슬라이스 대입은 삭제 후 삭제된 곳에 삽입하는 것과 비슷하다.
- 기본 슬라이스 S[i:j]에 대입된 반복은 크기가 같을 필요가 없다.
- 확장 슬라이스 S[i:j:k]에 대입된 반복은 크기가 반드시 같아야 한다.


#### 표4 가변 시퀀스 연산(리스트, bytearray)

| 연산자     | 설명     | 클래스 메서드 |
| ------ | ----------------- | ---------|
|S[i] = X |인덱스 대입: 오프셋 i에 있는 아이템을 X 참조로 변경| `__setitem__`|
|S[i:j] = I, S[i:j:k] = I | 슬라이스 대입: S의 k 간격(생략 가능)을 가진 i부터 j-1까지를 반복 가능한 I의 모든 아이템으로 대체|`__setitem__`|
|del S[i] | 인덱스 삭제 |`__delitem__`|
|del S[i:j], del S[i:j:k] | 슬라이스 삭제 |`__delitem__`|

### 파이썬 내장 자료형

#### 문자열
> 보통의 str 문자열 객체는 오프셋(위치)에 의해 접근되는 문자의 불변 시퀀스이다. 그것의 문자들은 기본 문자 집합의 코드 포인트 서수이며 각 문자는 길이가 1인 문자열 객체이다. 전체 문자열 객체 모델은 다양하며, 파이썬 3.X는 유사한 인터페이스의 세 가지 문자열 자료형을 가진다. 

- str
문자의 불변 시퀀스이며, ASCII와 고급 유니코드를 포함하는 모든 종류의 텍스트에 사용된다.
- bytes
short형 정수의 불변 시퀀스이며, 이진 데이터의 바이트 값을 위해 사용된다.
- bytearray
바이트의 가변형이다.


#### bytes, bytearray 가 문자열인 이유
>**`bytes`와 `bytearray`를 “문자열 자료형(string types)”이라고 부르는 이유는 “문자로 보이느냐”가 아니라, “문자열과 동일한 **시퀀스 모델과 API를 공유하기 때문**”입니다.**

***

1. 파이썬에서 말하는 “문자열”의 의미가 넓다

일반적으로 우리는 문자열 = **텍스트(str)** 라고 생각하지만,  
파이썬 내부 문서에서의 *string type*은:

> **“선형(linear)으로 나열된 요소(sequence)이며, 인덱싱·슬라이싱·반복 등의 문자열 연산을 지원하는 자료형”**

이라는 의미로 사용됩니다.

그래서 파이썬은 **3가지 문자열 계열(string-like types)** 을 둡니다.

| 타입          | 요소          | 목적         |
| ----------- | ----------- | ---------- |
| `str`       | 유니코드 코드 포인트 | 사람을 위한 텍스트 |
| `bytes`     | 0\~255 정수   | 이진 데이터     |
| `bytearray` | 0\~255 정수   | 가변 이진 데이터  |

***

2. 왜 `bytes`가 문자열 계열인가?

① 표현 방식이 “문자열 모델”을 그대로 따름

`bytes`는 **바이트열(byte sequence)** 이지만, 동작은 문자열과 거의 같습니다.

```python
b = b"ABC"

b[0]        # 65
b[0:2]      # b'AB'
len(b)      # 3
for x in b:
    print(x)
```

✅ **인덱싱, 슬라이싱, 반복, 길이**  
✅ `+`, `*`, `in`, `startswith`, `endswith`

이건 `str`과 **완전히 동일한 시퀀스 인터페이스**예요.

👉 파이썬은 “문자열처럼 다루는 연속 데이터”를 한 범주로 묶습니다.

#### 리터럴과 생성
>문자열 리터럴은 따옴표 안에 문자를 쓰는 방식으로 사용하며 선행되는 지정자 문자는 생략 가능하다. 그리고 모든 문자열 리터럴 형식에서 빈 문자열은 인접한 두 따옴표로 코딩된다. 다음의 다양한 내장 연산자들 또한 새 문자열을 반환한다.

- 'Pythons"s', "Python's"
> 작은따옴표와 큰따옴표는 동일하게 동작한다. 큰따옴표는 이스케이프되지 않은 작은따옴표를, 작은따옴표는 이스케이프되지 않은 큰따옴표를 내장할 수 있다.

- """This is a multiline block"""
> 삼중 따옴표 블록은 여러 라인의 텍스트를 한 문자열로 모은다. 
end-of-line 표시(\n)는 원래 인용 라인의 사이에 포함된다.

- 'Python\'s\n'
>백슬래시 이스케이프 코드 시퀀스는 그것이 나타내는 특수 문자 코드 포인트 값으로 바뀐다(예: '\n'은 10진수 코드 포인트 값이 10인 ASCII 문자이다).

- 당신이 'Python\n'이라는 문자열을 정의하면, 파이썬 인터프리터는 이 문자열을 메모리에 저장할 때 \n을 **실제 개행 문자 (코드 포인트 10)**로 변환합니다.
- 결과적으로, 이 문자열을 출력(print)하면 "Python"이라는 글자가 나오고, 그 다음에 \n이 있기 때문에 다음 줄로 커서가 이동한 후 출력 종료됩니다.

다른 이스케이프 시퀀스 예시:
- \t: 탭 (Tab)
- \\: 백슬래시 자체
- \': 작은따옴표 (문자열이 작은따옴표로 시작할 때 문자열 내부에 작은따옴표를 넣고 싶을 때)
- \": 큰따옴표
- \ooo: 8진수 값으로 문자 표현 (예: \10은 개행 문자를 8진수로 표현한 것)
- \xhh: 16진수 값으로 문자 표현 (예: \x0A는 개행 문자를 16진수로 표현한 것)

- "This" "is" "concatenated"
> 인접한 문자열 상수들은 연결된다. 이 형식을 괄호로 싸면 어려 줄에 걸쳐 표현할 수 있다.

- r'a raw\string', R'another\one'
> 원시 문자열: 백슬래시는 문자열의 끝인 경우를 제외하면 문자 그대로 유지된다. 예를 들어 r'c:\dir1\file'과 같이 정규식이나 윈도우(DOS) 디렉토리 경로를 나타내는 데 유용하다.
- 원시 문자열 안에서는 백슬래시(\)가 특별한 의미를 가지지 않고, 그 자체로 평범한 문자 백슬래시로 취급됩니다.
- 따라서 r'a raw\string'이라는 문자열을 정의하면, \s는 이스케이프 시퀀스로 해석되지 않고 \와 s라는 두 개의 문자로 인식됩니다.
- 예외: "문자열의 끝인 경우를 제외하면"이라는 단서가 붙어 있습니다. 이는 r'abc\'와 같이 원시 문자열의 맨 끝에 백슬래시가 오는 경우 구문 분석 오류가 발생할 수 있기 때문입니다. (백슬래시가 따옴표를 이스케이프하려 시도하는 것처럼 보일 수 있기 때문인데, 이런 경우는 거의 없습니다.)

>일반 문자열에서는 백슬래시(\) 뒤에 특정 문자가 오면 이스케이프 시퀀스로 해석되어 특수한 의미를 가집니다. 하지만 원시(raw) 문자열은 이러한 이스케이프 시퀀스 해석을 비활성화합니다.

원시 문자열의 정의:
- 문자열 앞에 r 또는 R 접두사를 붙여서 만듭니다 (예: r'hello' 또는 R"world").
- 이 r은 "raw"를 의미하며, "있는 그대로의 문자열"이라는 뜻입니다.

언제 유용한가?
*   **정규식 (Regular Expressions):** 정규식은 `\d` (숫자), `\w` (단어 문자) 등 백슬래시를 포함하는 특별한 패턴 문법을 많이 사용합니다. 일반 문자열로 정규식을 작성하면 `\\d`와 같이 백슬래시를 두 번 써야 하는 불편함이 있습니다. 원시 문자열을 사용하면 `r'\d'`와 같이 편리하게 작성할 수 있습니다.
    ```python
    import re
    # 일반 문자열: 백슬래시를 두 번 써야 \d가 됩니다.
    print(re.search('\\d+', 'abc123def')) # <re.Match object; span=(3, 6), match='123'>
    # 원시 문자열: 백슬래시를 한 번만 써도 됩니다.
    print(re.search(r'\d+', 'abc123def')) # <re.Match object; span=(3, 6), match='123'>
    ```
*   **파일 경로 (특히 Windows):** 윈도우 운영체제에서는 디렉토리 경로를 나타낼 때 백슬래시를 구분자로 사용합니다 (예: `C:\Users\Documents`). 일반 문자열로 이를 표현하려면 `C:\\Users\\Documents`처럼 백슬래시를 두 번씩 써야 합니다. 원시 문자열을 사용하면 `r'C:\Users\Documents'`와 같이 직관적으로 작성할 수 있습니다.
    ```python
    # 일반 문자열: 백슬래시 두 번
    path_normal = 'C:\\Program Files\\My App\\data.txt'
    print(path_normal)
    # 출력: C:\Program Files\My App\data.txt

    # 원시 문자열: 백슬래시 한 번
    path_raw = r'C:\Program Files\My App\data.txt'
    print(path_raw)
    # 출력: C:\Program Files\My App\data.txt
    ```

#### 문자열 포매팅
> 파이썬 3.X와 2.X에서 (3.0과 2.6) 일반 str 문자열은 포맷 서술 문자열에 따라 객체를 포맷하는 문자열 포매팅 연산에 두 가지 다른 방식을 지원한다.

- (모든 파이썬 버전에서) 원래부터 사용하던 표현식이며, % 연산자로 코딩된다: fmt % (values)
- (3.0, 2.6이후) 새로 지원되는 메서드이며, 호출 문법으로 코딩된다: fmt.format(values)

```python
'%s, %s, %.2f' % (42, 'spam', 1/3.0)

>> '42, spam, 0.33'

'{0}, {1}, {2:.2f}'.format(42, 'spam', 1 / 3.0)

>> '42, spam, 0.33'
```

#### 문자열 포매팅 표현식
> 문자열 포매팅 표현식은 % 연산자 왼쪽 문자열 안의 % 대상들을 % 연산자 오른쪽 값으로 대체한다(C의 sprintf와 유사). 만약 값을 두 개 이상 대체해야 한다면, 그것들은 % 연산자 오른쪽에 튜플로 코딩되어야 한다.
만약 대체할 값이 단지 한 개뿐이면 그것은 오른쪽에 단일 값이나 단일 아이템 튜플로 코딩될 수 있다(튜플 자체를 포맷하기 위한 중첩 튜플). 왼쪽에 키 이름이 사용되면 오른쪽에는 반드시 딕셔너리가 제공되야 한다. 그리고 *는 동적으로 너비와 정밀도를 전달할 수 있도록 허용한다.

```python
'The knights who say %s!' % 'Ni'
>> 'The knights who say Ni!'

'%d %s, %d you' % (1, 'spam', 4.0)
>> '1 spam, 4 you'

'%(n)d named %(x)s' % {'n': 1, 'x': "spam"}
>> '1 named spam'

'%(n).0E => [%(x)-6s]' % dict(n=100, x='spam')
>> '1E+02 => [spam ]'
```

#### 포매팅 표현식 문법
> % 연산자 왼쪽의 포맷 문자열에서 치환 대상은 다음의 일반 포맷을 가진다. 마지막 요소를 제외한 나머지는 생략 가능하다(치환 대상 바깥의 텍스트는 그대로 유지된다).

`%[(keyname)][flags][width][.prec]typecode`

- keyname
    - 괄호 안의 예상 딕셔너리 아이템을 참조한다.
- flags
    - -(왼쪽 맞춤), +(수치 기호), 스페이스(양수 앞에는 빈칸을 사용하고 음수 앞에는 -를 사용), 0(0으로 채움)
- width
    - 총 최소 필드 너비(값으로부터 가져오기 위해서 * 사용)
- prec
    - . 이후에 올 숫자 개수(값으로부터 가져오기 위해서 * 사용)
- typecode
    - 표 8의 문자

> 런타임까지 크기를 알 수 없을 경우, width와 prec는 % 연산자 오른쪽의 다음 아이템으로부터 값을 얻을 수 있도록 *로 코딩될 수 있다. %s는 일반적으로 어떤 객체 자료형이든 그것의 출력 대표 문자열로 반환한다.

#### 표8 % 문자열 포매팅 자료형 코드

| 코드 | 의미                    |
| -------- | --------------------- |
| `s`    | 문자열(혹은 str()을 사용하는 임의의 객체)    |
| `r`  | str()이 아닌 repr()을 사용하는 s |
| `c` | 문자(int 혹은 str)|
| `d` | 10진수 |
| `i` | 정수 |
| `u` | d와 동일(구식) |
| `o` | 8진수 |
| `x` | 16진수 |
| `X` | 대문자로 된 x |
| `e` | 부동 소수점 지수 |
| `E` | 대문자로 된 e |
| `f` | 부동 소수점 10진수 |
| `F` | 대문자로 된 f | 
| `g` | 부동 소수점 e 혹은 f |
| `G` | 부동 소수점 E 혹은 F | 
| `%` | 리터럴 '%'(%%로 코드) |

#### 문자열 포매팅 메서드
> 포매팅 메서드 호출은 이전 절의 표현식과 유사하다. 그러나 그것은 % 대신 {} 문법으로 치환 대상을 식별하는 포맷 문자열 객체상의 일반적인 메서드 호출을 통해 동작한다.

> 포맷 문자열 안의 치환 대상은 위치나 키워드명으로 메서드 호출 인자를 지정할 수 있고 레퍼런스 인자 속성, 키, 오프셋을 진행시킬 수 있다. 또는 기본 포매팅을 수용하거나 명시적인 자료형 코드를 제공할 수 있고, 인자 리스트로부터 값을 가져오기 위해 대상 문법을 중첩할 수 있다.

> 대부분의 `format` 메서드 애플리케이션은 앞의 절에서 본 % 표현식 사용 패턴과 동일한 기능을 가지고 있다. 그러나 메서드는 일부 연산자들이 포맷 문자열 안에 코딩되도록 허용한다.

```python
import sys # 메서드 vs 표현식:속성, 키, 인덱스

fmt = '{0.platform} {1[x]} {2[0]}'
fmt.format(sys, dict(x='ham'), 'AB')
>> 'win32 ham A'

fmt = '%s %s %s'
fmt % (sys.platform, dict(x='ham')['x'], 'AB'[0])
>> 'win32 ham A'
```

#### 리스트
> 리스트는 오프셋(위치)에 의해 접근되는 가변(변경 가능한) 객체 참조 시퀀스다.

- 리터럴과 생성
> 리스트 리터럴은 대괄호 안에 쉼표로 구분된 일련의 값으로 쓰이며, 다양한 연산들이 동적으로 리스트를 생성한다.

`L = [x ** 2 for x in range(9)]` : 반복 중인 표현식 결과를 수집하여 리스트를 생성한다(리스트 컴프리헨션).

#### 리스트 컴프리헨션 표현식
> 대괄호에 둘러싸인 리스트 리터럴([...])은 표현식의 간단한 리스트거나 다음과 같은 리스트 컴프리헨션 표현식일 수 있다.

```
[expr for target1 in iterable1 [if condition1]
      for target2 in iterable2 [if condition2] ... 
      for targetN in iterableN [if conditionN] ]
```

#### 반복 프로토콜
> 반복 프로토콜은 컬렉션의 아이템들이나 즉시 생성된 결과들을 자동으로  둘러보기 위해 모든 반복 컨텍스트가 사용하는 객체나 메서드의 집합을 정의하는데, 이는 컴프리헨션, for 루프문, map()과 filter() 같은 내장 함수를 포함한다. 반복은 다음과 같이 동작한다.

- 반복 컨텍스트는 `__iter__()` 메서드를 가지는 반복 객체에서 동작한다.
- 호출 시, 반복 객체의 `__iter__()` 메서드는 `__next__()` 메서드를 가지는 객체인 이터레이터를 반환한다(아마도 같은 객체).
- 호출 시, 반복 객체의 `__next__()` 메서드는 반복 중 다음 아이템을 반환하거나 반복을 끝내기 위해 StopIteration 예외를 발생시킨다.

#### 제너레이터 표현식
> 파이썬 2.4부터, 제너레이터 표현식은 모든 결과를 유지하기 위해 물리적인 리스트를 생성하지 않는 리스트 컴프리헨션과 비슷한 효과를 얻는다. 제너레이터 표현식은 결과 집합을 정의하지만, 메모리 절약을 위해 전체 리스트를 구성하지는 않는다. 그 대신에 제너레이터 표현식은 이전 절에 나온 반복 프로토콜을 자동으로 지원하는 반복 컨텍스트의 요소를 하나씩 반환하는 제너레이터 객체를 생성한다. 예를 들면 다음과 같다.

```
ords = (ord(x) for x in aString if x not in skipStr)
for o in ords:
    ...
```

> 제너레이터 표현식은 대괄호보다는 소괄호로 코딩되는 컴프리헨션이지만, 그 이외의 모든 리스트 컴프리헨션 문법을 지원한다. 함수에 전달될 반복을 생성할 때 단일 인자 함수를 위한 괄호를 사용하면 된다.

```
sum(ord(x) for x in aString)
```

> 파이썬 2.X 혹은 3.X 모두 제너레이터 표현식 루프 변수(예: 앞 예시의 x)는 제너레이터 표현식 밖에서 접근할 수 없다. 2.X에서 리스트 컴프리헨션은 루프 변수를 그것의 마지막 값으로 대입되도록 허용한다. 그러나 모든 컴프리헨션들은 변수를 표현식 내부로 지역화한다. 파이썬 3.X에서 루프 변수는 모든 컴프리헨션 형식에서 표현식 내부로 지역화된다.

```python
# 제너레이터 표현식
squares = (x ** 2 for x in range(5))
# 여기서 x는 제너레이터 표현식 내부에서만 유효합니다.
# 제너레이터 표현식 밖에서는 x에 접근할 수 없습니다.
# print(x) # -> NameError: name 'x' is not defined

# 리스트 컴프리헨션 (Python 3.x 기준)
my_list = [y * 2 for y in range(3)]
# 여기서 y는 리스트 컴프리헨션 내부에서만 유효합니다.
# print(y) # -> NameError: name 'y' is not defined
```

파이썬 2.x와 3.x의 차이:
- 파이썬 2.x: 리스트 컴프리헨션의 루프 변수는 전역 스코프에 영향을 미쳤습니다. (즉, 컴프리헨션 밖에서도 마지막 값으로 접근 가능했음). 이건 혼란을 야기하는 부분이 많아서 좋지 않은 디자인으로 평가받았습니다.
- 파이썬 3.x: 이런 문제점을 개선하여 모든 컴프리헨션(리스트, 딕셔너리, 셋, 제너레이터 표현식)의 루프 변수는 항상 표현식 내부로 '지역화'됩니다. (컴프리헨션 밖에서는 접근 불가).


> for 루프처럼 반복 컨텍스트 외부 결과로 진입하기 위해서 3.X의 I.__next__() 메서드(파이썬 2.X에서는 I.next() 메서드)나 파이썬 2.X나 3.X 모두에서 적절한 메서드를 호출해 주는 next(I) 내장 함수를 사용한다. 필요할 경우 모든 (남은) 결과를 한번에 산출하기 위해 `list()` 호출을 사용한다(제너레이터는 자체 이터레이터이므로 자체 `__iter__()`를 호출해도 무방하지만 불필요하다)

```python
squares = (x ** 2 for x in range(5))
squares
>> <generator object <genexpr> at 0x027C1AF8>

iter(squares) is squares # 선택적 __iter__() 가능
>> True

squares.__next__() # 메서드
>> 0

next(squares)   # 내장 (3.X, 2.6+)
>> 1

list(squares) # StopIteration까지
>> [4, 9, 16]
```

#### 제너레이터 표현식이란?
>제너레이터 표현식을 이해하기 위해서는 먼저 "리스트 컴프리헨션"과 "제너레이터"라는 개념을 알아야 합니다.

1.  **리스트 컴프리헨션 (List Comprehension) - "미리 만들어 놓은 뷔페 음식"**
    *   이것은 파이썬에서 리스트를 아주 간결하게 만드는 방법입니다.
    *   예를 들어, `[x * 2 for x in range(5)]` 라고 하면 `[0, 2, 4, 6, 8]` 이라는 리스트가 **한 번에 통째로** 만들어집니다.
    *   **단점**: 만약 리스트의 크기가 엄청나게 크다면 (예: 1억 개의 요소), 이 모든 요소를 한 번에 만들어서 메모리에 저장해야 하기 때문에 **메모리를 많이 잡아먹고** 느려질 수 있습니다. 마치 뷔페 음식을 손님이 오기도 전에 수백 가지를 미리 다 만들어 놓는 것과 같아요.

2.  **제너레이터 (Generator) - "주문하면 그때그때 요리해주는 식당"**
    *   제너레이터는 리스트처럼 모든 것을 한 번에 만들지 않고, **필요할 때마다 하나씩 값을 생성**해주는 특별한 함수(또는 객체)입니다.
    *   `yield` 키워드를 사용해서 만듭니다.
    *   **장점**: 모든 값을 메모리에 저장하지 않으므로 **메모리를 효율적으로 사용**할 수 있습니다. 예를 들어, 무한한 숫자 시퀀스를 다룰 때 유용합니다. 손님이 주문하면 그때그때 신선하게 요리해서 내주는 식당과 같습니다.


>**제너레이터 표현식은 리스트 컴프리헨션처럼 간결하게 코드를 작성하면서, 제너레이터의 장점(메모리 효율성)을 동시에 얻을 수 있는 방법입니다.**

>쉽게 말해, **"리스트 컴프리헨션처럼 보이지만, 실제로는 제너레이터처럼 작동하는 것"** 입니다.

*   **리스트 컴프리헨션**은 대괄호 `[]`를 사용합니다. (예: `[x * 2 for x in range(5)]`)
*   **제너레이터 표현식**은 소괄호 `()`를 사용합니다. (예: `(x * 2 for x in range(5))`)

**핵심 차이:**

*   **리스트 컴프리헨션 (`[]`)**: 모든 결과를 즉시 계산해서 리스트를 통째로 메모리에 만듭니다. (미리 만들어 놓은 뷔페 음식)
*   **제너레이터 표현식 (`()`)**: 필요한 값을 **하나씩만** 생성하며, 모든 결과를 메모리에 미리 저장하지 않습니다. (주문하면 그때그때 요리해주는 식당)

**예시를 다시 봅시다:**

```python
ords = (ord(x) for x in aString if x not in skipStr)
for o in ords:
    print(o)
```

이 코드가 하는 일은 다음과 같습니다.

1.  `aString`이라는 문자열에서 `skipStr`에 없는 문자들만 골라냅니다.
2.  각 문자의 아스키 코드(`ord(x)`)를 계산합니다.
3.  **중요**: 이 모든 아스키 코드들을 **즉시 리스트로 만들어서 `ords`에 저장하지 않습니다.**
4.  대신 `ords`는 "나중에 `for` 루프가 돌 때마다 아스키 코드를 하나씩 생성해 줄게!"라고 약속하는 **제너레이터 객체**가 됩니다.
5.  `for o in ords:` 루프가 실행될 때, `ords`는 필요할 때마다 아스키 코드를 하나씩 만들어서 `o`에 넘겨줍니다.

**무슨 이점이 있나요?**

*   `aString`이 아주 긴 문자열이어도, 모든 아스키 코드 값을 한꺼번에 메모리에 저장할 필요가 없어 **메모리 사용량이 훨씬 적습니다.**
*   `for` 루프가 다 돌기도 전에 중단되면, 남은 값들은 아예 계산되지 않으므로 **성능 면에서도 이점**이 있을 수 있습니다.

**또 다른 예시:**

```python
sum(ord(x) for x in aString)
```

이 코드는 `aString`에 있는 모든 문자의 아스키 코드 합을 계산합니다.
여기서 `(ord(x) for x in aString)` 부분이 제너레이터 표현식입니다.
`sum()` 함수는 이 제너레이터 표현식으로부터 값을 하나씩 받아서 더해나가기 때문에, 역시 모든 아스키 코드 값을 미리 리스트로 만들 필요가 없습니다.

**정리:**

*   **제너레이터 표현식은 리스트 컴프리헨션의 편리함(짧은 코드)과 제너레이터의 효율성(적은 메모리 사용)을 합친 강력한 기능입니다.**
*   대괄호 `[]` 대신 소괄호 `()`를 사용합니다.
*   값을 한꺼번에 만들어 리스트에 저장하는 대신, **필요할 때마다 하나씩 생성**하여 메모리를 절약합니다.

#### 이터러블(iterable)과 이터레이터(iterator)

>파이썬에서 `for` 루프를 돌릴 수 있는 객체는 두 가지 개념으로 나뉩니다.

1.  **이터러블 (Iterable):**
    *   `for` 루프를 돌 수 있는 **능력이 있는** 객체입니다. (예: 리스트, 튜플, 문자열, 딕셔너리, 제너레이터 표현식)
    *   `__iter__()` 라는 특별한 메서드를 가지고 있어서, 이 메서드를 호출하면 **이터레이터**를 반환합니다.
    *   비유: **뷔페 음식 진열대**라고 생각해보세요. 여기저기 많은 음식이 놓여있고, 여기서 음식을 집어가기 위한 "접시"를 만들 수 있습니다.

2.  **이터레이터 (Iterator):**
    *   실제로 `for` 루프를 돌면서 **한 번에 하나씩 값을 만들어내는** 객체입니다.
    *   `__next__()` (파이썬 3.x) 또는 `next()` (파이썬 2.x)라는 특별한 메서드를 가지고 있어서, 이 메서드를 호출할 때마다 다음 값을 반환합니다. 더 이상 값이 없으면 `StopIteration` 예외를 발생시킵니다.
    *   `__iter__()` 메서드도 가지고 있는데, 자신(`self`)을 반환합니다.
    *   비유: 뷔페 음식 진열대에서 음식을 집어 담기 시작한 **"접시"**라고 생각해보세요. 이 접시로 음식을 하나씩 집어와서 먹을 수 있습니다.

**제너레이터의 특징:**

제너레이터는 아주 특별하게도 **이터러블인 동시에 이터레이터**입니다.

*   제너레이터 객체는 `for` 루프를 돌 수 있으니 **이터러블**입니다.
*   제너레이터 객체는 `__next__()` 메서드를 가지고 있어서 다음 값을 생성할 수 있으니 **이터레이터**입니다.

**`iter(squares) is squares` 설명:**

```python
squares = (x ** 2 for x in range(5))
# squares는 제너레이터 객체입니다.
# 즉, 이터러블이면서 동시에 이터레이터입니다.

# iter() 함수는 이터러블 객체에서 이터레이터를 얻는 함수입니다.
# squares는 이터러블이므로 iter(squares)를 호출하면 이터레이터를 반환해야 합니다.

# 그런데 squares는 이미 그 자체로 이터레이터입니다!
# 그래서 iter(squares)를 호출하면 자기 자신(squares)을 그대로 반환합니다.

# 'is' 연산자는 두 객체가 완전히 동일한 객체인지(메모리 주소가 같은지) 확인합니다.
iter(squares) is squares
# 결과: True
# 이것은 iter(squares)가 squares와 완전히 동일한 객체라는 것을 의미합니다.
# 즉, squares가 이미 이터레이터라는 것을 확인시켜 줍니다.
```

**`# 선택적 __iter__() 가능` 이 무슨 말인가요?**

일반적으로 이터러블은 `__iter__()` 메서드를 구현하여 "새로운" 이터레이터 객체를 반환합니다. 하지만 이터레이터 객체는 `__iter__()` 메서드를 구현할 때 자기 자신(`self`)을 반환하도록 되어 있습니다.

제너레이터는 이미 이터레이터이므로, 그 내부에 `__iter__()`가 자기 자신을 반환하도록 되어 있습니다. 그래서 `iter(squares)`를 호출하면 `squares` 객체의 `__iter__()` 메서드가 호출되고, 이 메서드는 자기 자신인 `squares`를 반환하는 것이죠. "선택적"이라는 표현은 이터레이터가 자신을 반환하는 `__iter__()`를 가질 수 있다는 의미로 쓰인 것으로 보입니다. (필수적으로 다른 이터레이터를 만들어 반환해야 하는 일반적인 이터러블과는 다르다는 의미)

**요약하자면:**

`iter(squares) is squares`는 **"제너레이터 객체 `squares`는 이터러블이면서 동시에 이터레이터이므로, `iter()` 함수에 전달해도 새로운 이터레이터 객체를 만들지 않고 자기 자신을 그대로 반환한다"**는 것을 보여주는 코드입니다.
